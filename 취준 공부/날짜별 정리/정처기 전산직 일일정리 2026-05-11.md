오늘 주제: SQL/DB - NOT IN과 NULL의 3값 논리

1. 핵심 요약
- NULL은 0이나 빈 문자열이 아니라 “알 수 없음/값 없음”을 뜻합니다.
- SQL에서는 TRUE, FALSE 외에 UNKNOWN까지 고려하는 3값 논리가 사용됩니다.
- `컬럼 = NULL`, `컬럼 <> NULL` 같은 비교는 TRUE/FALSE가 아니라 UNKNOWN이 됩니다.
- `IN`은 값 목록 중 하나와 일치하는지 확인하는 조건입니다.
- 반대로 `NOT IN`은 목록의 모든 값과 일치하지 않아야 TRUE가 됩니다.
- 그런데 `NOT IN` 목록이나 서브쿼리 결과에 NULL이 섞이면 비교 결과가 UNKNOWN이 되어 예상한 행이 빠질 수 있습니다.
- SQLD에서도 NULL, IN/NOT IN, EXISTS/NOT EXISTS 차이는 자주 보는 포인트입니다.
- NULL 가능성이 있는 서브쿼리에서는 `WHERE 컬럼 IS NOT NULL`로 NULL을 제거하거나 `NOT EXISTS`를 고려하는 방식이 안전합니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: NULL 비교는 `= NULL`이 아니라 `IS NULL`, `IS NOT NULL`을 사용합니다.
- SQLD 포인트: `NOT IN` 서브쿼리 결과에 NULL이 있으면 결과가 예상과 다를 수 있습니다.
- 비교 포인트: `NOT IN`은 값 비교 중심, `NOT EXISTS`는 조건을 만족하는 행의 존재 여부 중심입니다.

3. OX 문제
SQL에서 NOT IN 서브쿼리 결과에 NULL이 포함되어도, 조건은 NULL을 제외한 나머지 값들만 비교해 예상대로 TRUE 또는 FALSE를 반환한다.

오늘 주제: SQL/DB - WHERE 조건 BETWEEN과 IN

1. 핵심 요약
- WHERE절은 테이블에서 조건에 맞는 행만 걸러내는 SQL의 기본 필터링 절입니다.
- BETWEEN A AND B는 값이 A 이상 B 이하 범위에 있는지 검사합니다.
- 중요한 점은 BETWEEN은 양 끝값을 포함한다는 것입니다.
- 예를 들어 점수 BETWEEN 80 AND 90이면 80점과 90점도 포함됩니다.
- IN은 값이 여러 후보 중 하나와 일치하는지 확인할 때 씁니다.
- deptno IN (10, 20, 30)은 deptno = 10 OR deptno = 20 OR deptno = 30과 비슷하게 이해할 수 있습니다.
- IN 안에는 직접 값 목록을 넣을 수도 있고, 서브쿼리 결과를 넣을 수도 있습니다.
- SQLD에서도 자주 보는 포인트는 BETWEEN의 포함 범위, IN과 OR의 관계, 그리고 NULL이 섞인 NOT IN의 주의점입니다.
- 특히 NOT IN 서브쿼리 결과에 NULL이 포함되면 비교 결과가 UNKNOWN이 되어 기대와 다른 결과가 나올 수 있습니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: BETWEEN은 양 끝값 포함, 즉 “이상·이하” 조건입니다.
- 정처기/SQLD 공통: IN은 여러 값 중 하나와 일치하는지 확인하며, OR 조건과 연결해서 이해하면 쉽습니다.
- SQLD 포인트: NOT IN과 NULL이 함께 나오면 3값 논리 때문에 결과가 비어 보일 수 있어 주의해야 합니다.

3. OX 문제
SQL의 BETWEEN은 양 끝값을 포함하는 범위 조건이며, IN은 나열된 값 목록 또는 서브쿼리 결과 중 하나와 일치하는지를 검사하는 조건이다.

오늘 주제: 네트워크 - IPsec AH와 ESP

1. 핵심 요약
- IPsec은 IP 계층에서 패킷을 보호하기 위한 보안 프로토콜 집합입니다.
- 즉, 특정 애플리케이션만 보호하는 것이 아니라 IP 패킷 단위로 보안을 적용할 수 있습니다.
- AH는 Authentication Header의 약자로, 주로 송신자 인증과 데이터 무결성 확인을 담당합니다.
- 무결성은 전송 중 데이터가 변조되지 않았는지 확인하는 성질입니다.
- ESP는 Encapsulating Security Payload의 약자로, 인증·무결성뿐 아니라 암호화를 통한 기밀성까지 제공할 수 있습니다.
- IPsec은 전송 모드와 터널 모드로 동작할 수 있습니다.
- 전송 모드는 원래 IP 헤더를 유지하고 주로 페이로드를 보호합니다.
- 터널 모드는 원래 IP 패킷 전체를 감싸 새 IP 헤더를 붙이는 방식이라 VPN과 자주 연결됩니다.
- 시험에서는 “IPsec = 네트워크 계층 보안”이라는 위치를 먼저 잡으면 AH/ESP 구분이 쉬워집니다.

2. 자주 나오는 포인트
- AH: 인증, 무결성 중심 / 기밀성 암호화는 핵심 기능이 아님
- ESP: 인증, 무결성, 암호화 가능 → 기밀성 제공 가능
- 터널 모드: 원래 IP 패킷 전체를 캡슐화하므로 VPN과 연관

3. OX 문제
IPsec은 네트워크 계층에서 IP 패킷을 보호하는 보안 프로토콜 집합이며, AH는 주로 인증과 무결성을, ESP는 인증·무결성뿐 아니라 암호화를 통한 기밀성까지 제공할 수 있다.

- 이 높이 차이를 균형 인수라고 하며, 보통 -1, 0, 1 범위를 유지해야 합니다.
- 삽입이나 삭제 후 균형이 깨지면 회전을 수행해 트리 높이를 다시 맞춥니다.
- 회전 유형은 LL, RR, LR, RL로 구분하며, 단일 회전 또는 이중 회전이 사용됩니다.
- 균형을 유지하기 때문에 탐색·삽입·삭제를 최악의 경우에도 O(log n) 수준으로 유지할 수 있습니다.

2. 자주 나오는 포인트
- AVL 트리 = 자가 균형 이진 탐색 트리입니다.
- 균형 인수는 왼쪽 서브트리와 오른쪽 서브트리의 높이 차이입니다.
- 일반 BST는 한쪽으로 치우치면 O(n), AVL은 회전으로 균형을 맞춰 O(log n)을 유지합니다.

3. OX 문제
AVL 트리는 이진 탐색 트리의 한 종류로, 삽입이나 삭제 후 균형 인수가 허용 범위를 벗어나면 회전을 통해 트리 높이를 조정한다.
