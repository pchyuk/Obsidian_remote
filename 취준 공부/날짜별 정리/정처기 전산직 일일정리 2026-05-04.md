오늘 주제: SQL/DB - SELECT DISTINCT와 중복 제거

1. 핵심 요약
- DISTINCT는 SELECT 결과에서 중복 행을 제거해 보여주는 키워드입니다.
- “테이블의 데이터를 삭제”하는 것이 아니라, 조회 결과에서만 중복을 하나로 줄입니다.
- `SELECT DISTINCT 컬럼명`은 해당 컬럼 값이 같은 행을 하나만 출력합니다.
- 여러 컬럼을 함께 쓰면 각 컬럼 값의 “조합”이 모두 같을 때 중복으로 봅니다.
- 예를 들어 `DISTINCT 부서, 직급`이면 부서만 같다고 중복이 아니라, 부서와 직급이 모두 같아야 중복입니다.
- NULL도 DISTINCT의 중복 판단 대상에 포함될 수 있어, 같은 컬럼 조합의 NULL 행은 하나로 줄어들 수 있습니다.
- SQLD에서도 SELECT 기본 문법에서 자주 보는 포인트이며, GROUP BY와 헷갈리지 않는 것이 중요합니다.
- GROUP BY는 그룹별 집계가 목적이고, DISTINCT는 단순 중복 제거가 목적입니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: DISTINCT는 원본 데이터를 바꾸지 않고 조회 결과의 중복만 제거한다.
- SQLD 포인트: 여러 컬럼에 DISTINCT를 쓰면 컬럼 하나가 아니라 컬럼 조합 전체로 중복을 판단한다.
- 주의: DISTINCT와 GROUP BY는 결과가 비슷해 보일 수 있지만, GROUP BY는 집계 함수와 함께 그룹 단위 분석에 자주 사용된다.

3. OX 문제
SELECT DISTINCT는 지정한 컬럼 조합이 중복되는 행을 하나로 줄이며, NULL 값도 중복 판단 대상에 포함될 수 있다.

오늘 주제: 네트워크 - CSMA/CD와 CSMA/CA

1. 핵심 요약
- CSMA는 여러 장비가 하나의 통신 매체를 공유할 때, 전송 전에 매체가 비어 있는지 확인하는 방식입니다.
- CSMA/CD의 CD는 Collision Detection, 즉 충돌 감지입니다.
- 유선 공유 이더넷 환경에서는 동시에 두 장비가 전송하면 충돌이 발생할 수 있고, 이를 감지하면 전송을 멈춘 뒤 랜덤 시간 후 재전송합니다.

- 핵심은 CD는 “충돌이 나면 감지하고 재전송”, CA는 “충돌이 나기 전에 피하려고 시도”입니다.

오늘 주제: 자료구조 - 퀵 정렬과 피벗 분할

1. 핵심 요약
- 퀵 정렬은 분할 정복 방식의 대표적인 정렬 알고리즘입니다.
- 먼저 기준값인 피벗을 하나 정하고, 피벗보다 작은 값과 큰 값을 나누어 배치합니다.

- 분할이 균형 있게 이루어지면 전체 정렬 시간은 평균적으로 O(n log n)에 가깝습니다.
- 하지만 피벗이 계속 최솟값이나 최댓값처럼 한쪽으로 치우치면 분할 효과가 떨어집니다.
- 이 경우 한쪽 부분 배열만 거의 n-1 크기로 남아 최악 시간복잡도는 O(n²)이 됩니다.
- 그래서 퀵 정렬은 평균적으로 빠르지만, 피벗 선택에 따라 성능 차이가 커질 수 있습니다.

2. 자주 나오는 포인트
- 퀵 정렬: 피벗 선택 → 분할 → 재귀 정렬 구조
- 평균 시간복잡도는 O(n log n), 최악 시간복잡도는 O(n²)
- 병합 정렬, 선택 정렬, 삽입 정렬과 시간복잡도 비교 문제로 자주 출제

오늘 주제: SQL/DB - 개체 무결성과 참조 무결성

1. 핵심 요약
