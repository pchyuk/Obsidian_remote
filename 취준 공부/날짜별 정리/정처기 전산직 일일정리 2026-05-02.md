오늘 주제: SQL/DB - 데이터 모델링 식별자와 주식별자

1. 핵심 요약
- 식별자는 엔터티 안의 여러 인스턴스를 서로 구분해 주는 속성 또는 속성 집합입니다.
- 예를 들어 학생 엔터티에서 학번이 각 학생을 유일하게 구분한다면 학번은 식별자가 될 수 있습니다.
- 후보 식별자는 유일성과 최소성을 만족해 식별자로 쓸 수 있는 후보들입니다.
- 그중 업무적으로 대표로 선택한 식별자를 주식별자라고 합니다.
- 관계형 모델로 바꾸면 주식별자는 보통 기본키(PK)가 됩니다.
- 선택되지 않은 후보 식별자는 대체키가 될 수 있습니다.
- 식별자는 가능하면 값이 자주 바뀌지 않아야 하고, NULL이 되면 안 됩니다.
- SQLD에서도 데이터 모델링 파트에서 엔터티, 속성, 관계와 함께 식별자 개념을 자주 묻습니다.
- 정처기에서는 기본키, 후보키, 대체키, 외래키와 연결해서 출제되는 경우가 많습니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: 주식별자 = 후보 식별자 중 대표로 선택한 식별자
- SQLD 포인트: 식별자는 유일성, 최소성, 불변성, 존재성을 만족하는지 확인
- DB 포인트: 주식별자는 테이블 변환 시 기본키가 되고, 관계 설정 시 외래키와 연결될 수 있음

3. OX 문제
데이터 모델링에서 식별자는 엔터티의 인스턴스를 유일하게 구분하는 속성 또는 속성 집합이며, 후보 식별자 중 대표로 선택된 것이 주식별자이다.

해설: 식별자는 엔터티 인스턴스, 즉 개별 행을 유일하게 구분하는 속성 또는 속성 집합입니다. 여러 후보 식별자 중 대표로 선택한 것이 주식별자이고, 관계형 모델에서는 보통 기본키(PK)로 이어집니다. SQLD/정처기 모두 식별자의 유일성, 최소성, 불변성, 존재성과 후보키·기본키·대체키 관계를 자주 묻습니다.

오늘 주제: 네트워크 - 스위치와 MAC 주소 테이블

1. 핵심 요약
- 스위치는 주로 데이터 링크 계층, 즉 2계층에서 동작하는 네트워크 장비입니다.
- 핵심 기준은 IP 주소가 아니라 MAC 주소입니다.
- 스위치는 프레임이 들어오면 출발지 MAC 주소와 들어온 포트를 MAC 주소 테이블에 기록합니다.
- 이후 목적지 MAC 주소가 테이블에 있으면 해당 포트로만 프레임을 전달합니다.
- 목적지 MAC 주소를 모르면 같은 VLAN 안의 다른 포트들로 프레임을 플러딩할 수 있습니다.
- 허브는 들어온 신호를 모든 포트로 단순 전달하지만, 스위치는 MAC 주소를 학습해 더 효율적으로 전달합니다.
- 그래서 스위치는 포트별 충돌 영역을 나누어 네트워크 효율을 높입니다.
- 라우터는 3계층에서 IP 주소와 라우팅 테이블을 보고 다른 네트워크로 패킷을 전달한다는 점에서 스위치와 구분됩니다.

2. 자주 나오는 포인트
- 스위치: 2계층, MAC 주소 기반, 프레임 전달
- 라우터: 3계층, IP 주소 기반, 서로 다른 네트워크 연결
- 목적지 MAC을 모르면 플러딩, 알면 해당 포트로 선택 전달

3. OX 문제
스위치는 MAC 주소 테이블을 이용해 목적지 MAC 주소가 연결된 포트로 프레임을 전달하며, 목적지 MAC을 모르면 플러딩할 수 있다.

1. 핵심 요약
- 교착상태는 여러 프로세스가 서로 자원을 기다리며 더 이상 진행하지 못하는 상태입니다.
- 처리 방법은 크게 예방, 회피, 탐지와 회복으로 나눕니다.
- 예방은 상호 배제, 점유와 대기, 비선점, 순환 대기 중 하나가 성립하지 않게 막는 방식입니다.
- 회피는 자원을 할당하기 전에 “이 할당이 위험한가?”를 판단해 안전한 경우에만 할당합니다.
- 대표적인 회피 기법이 은행원 알고리즘입니다.
- 은행원 알고리즘은 현재 가용 자원, 각 프로세스의 최대 요구량, 이미 할당된 자원, 추가 필요량을 보고 안전 상태 여부를 판단합니다.
- 안전 상태란 모든 프로세스가 어떤 순서로든 필요한 자원을 받아 종료될 수 있는 순서가 존재하는 상태입니다.
- 탐지와 회복은 교착상태 발생 가능성을 허용한 뒤, 실제 발생하면 찾아서 프로세스 종료나 자원 선점으로 해결하는 방식입니다.

2. 자주 나오는 포인트
- 예방: 교착상태 발생 조건 4가지 중 하나를 깨는 방식
- 회피: 안전 상태를 유지하도록 자원 할당 여부를 미리 판단
- 은행원 알고리즘: 교착상태 회피 기법, 안전 순서 존재 여부가 핵심
