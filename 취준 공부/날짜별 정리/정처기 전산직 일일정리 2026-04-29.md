1. 핵심 요약
- 윈도우 함수는 행들을 일정 기준으로 바라보며 순위, 누적합, 평균 같은 분석 값을 계산하는 SQL 기능입니다.
- 핵심은 `OVER()` 절입니다. 이 절 안에서 어떤 범위의 행을 기준으로 계산할지 정합니다.
- `PARTITION BY`는 계산 대상을 그룹처럼 나누는 역할을 합니다. 예를 들어 부서별 순위를 매길 수 있습니다.
- `ORDER BY`는 각 파티션 안에서 순서를 정합니다. 순위 함수나 누적합 계산에서 특히 중요합니다.
- `GROUP BY`는 여러 행을 하나의 그룹 결과로 줄이지만, 윈도우 함수는 원래 행을 유지하면서 계산 결과 컬럼을 추가합니다.
- 대표 함수에는 `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`, `SUM() OVER`, `AVG() OVER` 등이 있습니다.
- SQLD에서도 `PARTITION BY`, `ORDER BY`, `RANK`와 `DENSE_RANK` 차이는 자주 보는 포인트입니다.
- 정처기에서는 “집계는 하되 행은 유지한다”는 특징을 이해하면 문제를 쉽게 풀 수 있습니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: 윈도우 함수는 `OVER()` 절과 함께 사용한다.
- SQLD 포인트: `RANK()`는 공동 순위 다음 순위를 건너뛰고, `DENSE_RANK()`는 건너뛰지 않는다.
- 비교 포인트: `GROUP BY`는 결과 행 수가 줄어들 수 있지만, 윈도우 함수는 각 행을 유지한다.

3. OX 문제
윈도우 함수는 GROUP BY처럼 행을 하나로 합쳐 결과 행 수를 줄이는 것이 아니라, 각 행을 유지한 상태에서 순위나 누적합 같은 분석 값을 계산할 수 있다.

오늘 주제: 네트워크 - CIDR과 사용 가능 호스트 수 계산

1. 핵심 요약
- CIDR은 IP 주소 뒤에 `/24`, `/26`처럼 붙는 표기로, 네트워크 부분이 몇 비트인지 나타냅니다.
- IPv4 주소는 전체 32비트이므로 `/26`이면 앞 26비트는 네트워크, 뒤 6비트는 호스트 부분입니다.
- 호스트 비트가 n개이면 전체 주소 수는 `2^n`개입니다.
- 일반적인 서브넷에서는 첫 주소는 네트워크 주소, 마지막 주소는 브로드캐스트 주소로 예약됩니다.
- 그래서 사용 가능한 호스트 수는 보통 `2^호스트비트 - 2`로 계산합니다.
- 예를 들어 `/26`은 호스트 비트가 `32 - 26 = 6`개이므로 전체 64개, 사용 가능 호스트는 62개입니다.
- `/24`는 호스트 비트 8개라서 `2^8 - 2 = 254개`가 됩니다.
- 시험에서는 CIDR 표기, 서브넷 마스크, 네트워크 주소/브로드캐스트 주소, 사용 가능 호스트 수를 함께 묻는 경우가 많습니다.

2. 자주 나오는 포인트
- CIDR `/숫자`는 네트워크 비트 수이고, 호스트 비트 수는 `32 - 숫자`입니다.
- 사용 가능 호스트 수는 일반적으로 `2^호스트비트 - 2`입니다.
- `/25`, `/26`, `/27`, `/28`처럼 자주 나오는 값은 빠르게 계산할 수 있게 익혀두면 좋습니다.

3. OX 문제
CIDR 표기 /26은 호스트 부분이 6비트이므로, 일반적인 IPv4 서브넷에서 사용 가능한 호스트 수는 2^6 - 2 = 62개이다.

오늘 주제: 네트워크 - TCP 3-way handshake

1. 핵심 요약
- TCP는 연결형 전송 계층 프로토콜입니다.
- 데이터를 보내기 전에 송신자와 수신자가 서로 통신 가능한 상태인지 확인합니다.

- 이 과정에서 양쪽은 초기 순서 번호를 교환해 데이터 순서 제어와 신뢰성 있는 전송의 기반을 마련합니다.

- TCP: 연결형, 신뢰성, 순서 보장, 흐름 제어, 혼잡 제어
- 3-way handshake 순서: SYN → SYN+ACK → ACK

오늘 주제: 운영체제 - 페이지 교체 알고리즘 FIFO와 LRU

1. 핵심 요약
- 가상메모리에서는 필요한 페이지만 메모리에 올리고, 부족하면 기존 페이지 중 하나를 내보냅니다.
- 이때 어떤 페이지를 교체할지 정하는 기준이 페이지 교체 알고리즘입니다.
- FIFO는 First In First Out으로, 메모리에 가장 먼저 들어온 페이지를 먼저 교체합니다.

- “최근에 안 쓴 페이지는 앞으로도 덜 쓸 가능성이 높다”는 지역성 원리를 활용합니다.

2. 자주 나오는 포인트
- FIFO: “먼저 들어온 페이지”를 교체한다.
- LRU: “가장 오래 사용되지 않은 페이지”를 교체한다.
- 페이지 부재 횟수 계산 문제에서는 적재 시점과 최근 사용 시점을 혼동하지 않는 것이 핵심입니다.
