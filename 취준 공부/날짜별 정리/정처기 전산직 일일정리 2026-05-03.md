1. 핵심 요약
- 포트 번호는 한 컴퓨터 안에서 어떤 애플리케이션이 데이터를 받을지 구분하는 번호입니다.
- IP 주소가 “어느 장비로 보낼까?”를 정한다면, 포트 번호는 “그 장비 안의 어떤 프로그램으로 보낼까?”를 정합니다.
- TCP와 UDP 모두 포트 번호를 사용하며, 전송 계층에서 다룹니다.
- 예를 들어 웹 서버는 보통 HTTP 80번, HTTPS 443번 포트를 사용합니다.
- DNS는 주로 53번 포트를 사용하고, DHCP는 서버 67번·클라이언트 68번을 사용합니다.
- 클라이언트가 서버에 접속할 때는 서버의 well-known port로 요청을 보내고, 클라이언트 쪽은 임시 포트를 사용하는 경우가 많습니다.
- 포트 번호 덕분에 하나의 IP를 가진 서버가 웹, DNS, 메일 같은 여러 서비스를 동시에 제공할 수 있습니다.
- 시험에서는 IP 주소와 포트 번호의 역할 구분, TCP/UDP와의 관계, 주요 포트 번호 암기가 자주 나옵니다.

2. 자주 나오는 포인트
- IP 주소: 호스트 식별 / 포트 번호: 프로세스 또는 서비스 식별
- HTTP 80, HTTPS 443, DNS 53, DHCP 67/68, FTP 20/21, SSH 22, SMTP 25는 자주 출제됩니다.
- 포트 번호는 네트워크 계층이 아니라 전송 계층의 개념입니다.

- SQLD에서도 자주 보는 포인트는 “인덱스는 검색을 빠르게 하지만, 저장 공간과 갱신 비용이 든다”는 균형 감각입니다.

오늘 주제: 운영체제 - 프로세스 상태 전이

1. 핵심 요약
- 프로세스는 실행 중인 프로그램이며, 운영체제는 각 프로세스의 상태를 관리합니다.
- 대표 상태는 생성(New), 준비(Ready), 실행(Running), 대기/봉쇄(Waiting/Blocked), 종료(Terminated)입니다.
- 준비 상태는 CPU만 할당받으면 바로 실행 가능한 상태입니다.
- 실행 상태는 실제로 CPU를 점유해 명령어를 수행하는 상태입니다.
- 준비 상태의 프로세스가 CPU를 받는 것을 디스패치(dispatch)라고 합니다.
- 실행 중인 프로세스가 입출력 작업을 요청하면 결과를 기다려야 하므로 대기 상태로 이동합니다.
- 입출력이 완료되면 곧바로 실행되는 것이 아니라 준비 상태로 돌아가 CPU 배정을 기다립니다.
- 선점형 스케줄링에서는 타임슬라이스가 끝나거나 우선순위가 높은 프로세스가 오면 실행 상태에서 준비 상태로 밀려날 수 있습니다.

2. 자주 나오는 포인트
- 준비 → 실행: CPU 할당, 디스패치
- 실행 → 대기: 입출력 요청, 이벤트 대기
- 대기 → 준비: 입출력 완료 후 CPU를 기다림

3. OX 문제
운영체제에서 준비 상태(Ready)의 프로세스는 CPU를 할당받으면 실행 상태(Running)로 전이될 수 있고, 실행 중 입출력을 요청하면 대기 상태(Blocked/Waiting)로 전이될 수 있다.

오늘 주제: SQL/DB - OUTER JOIN과 NULL 확장

1. 핵심 요약
- OUTER JOIN은 조인 조건에 맞지 않는 행도 결과에 포함할 수 있는 조인입니다.
- LEFT OUTER JOIN은 왼쪽 테이블의 모든 행을 기준으로 결과를 만듭니다.

- INNER JOIN은 양쪽 테이블에서 조건이 맞는 행만 남기므로, 조건에 맞지 않는 행은 사라집니다.

- 조인 후 WHERE절에서 오른쪽 테이블 컬럼에 조건을 잘못 걸면 OUTER JOIN이 INNER JOIN처럼 동작할 수 있어 주의해야 합니다.
- 시험에서는 “누락된 데이터까지 포함”, “매칭되지 않는 행 보존”, “NULL 채움”이라는 표현이 나오면 OUTER JOIN을 떠올리면 좋습니다.

- 정처기/SQLD 공통: INNER JOIN은 조건 만족 행만, OUTER JOIN은 한쪽 또는 양쪽의 미매칭 행도 보존합니다.
- LEFT OUTER JOIN: 왼쪽 테이블 행은 모두 남고, 오른쪽에 매칭이 없으면 NULL이 들어갑니다.
