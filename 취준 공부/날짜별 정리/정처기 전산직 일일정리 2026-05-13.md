1. 핵심 요약

- ON DELETE CASCADE는 부모 행이 삭제되면 그 부모를 참조하는 자식 행도 함께 삭제되도록 합니다.

오늘 주제: 네트워크 - DHCP DORA 과정과 IP 임대

1. 핵심 요약
- DHCP는 클라이언트에게 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 주소 등을 자동으로 할당해 주는 프로토콜입니다.
- 수동으로 IP를 설정하지 않아도 네트워크에 접속할 수 있게 해 주므로, 관리 부담을 줄여 줍니다.
- DHCP는 주소를 영구히 주는 것이 아니라 일정 시간 동안 “임대”하는 방식으로 관리합니다.
- 대표 동작 흐름은 DORA: Discover → Offer → Request → ACK입니다.
- Discover는 클라이언트가 “DHCP 서버 있나요?” 하고 찾는 단계입니다.
- Offer는 DHCP 서버가 “이 IP를 써도 됩니다” 하고 제안하는 단계입니다.
- Request는 클라이언트가 제안받은 IP를 사용하겠다고 요청하는 단계입니다.
- ACK는 서버가 최종 승인하여 IP 임대를 확정하는 단계입니다.
- DHCP 서버는 보통 UDP 67번, 클라이언트는 UDP 68번 포트를 사용합니다.
- DNS가 도메인 이름을 IP 주소로 바꾸는 역할이라면, DHCP는 네트워크 설정값을 자동 배정하는 역할입니다.

2. 자주 나오는 포인트
- DHCP = IP 자동 할당 + 임대(Lease) 개념.
- DORA 순서: Discover → Offer → Request → ACK.
- DHCP는 DNS와 다릅니다. DHCP는 주소 설정 배정, DNS는 이름 해석입니다.

3. OX 문제
DHCP의 DORA 과정은 클라이언트가 Discover로 서버를 찾고, 서버가 Offer를 제안한 뒤, 클라이언트가 Request로 선택을 알리고, 서버가 ACK로 임대를 확정하는 흐름이다.

1. 핵심 요약

- 체크섬은 오류를 “검출”하는 기능이지, 오류 난 비트를 직접 고쳐 주는 “수정” 기능은 아닙니다.

- UDP에도 체크섬이 있지만, UDP는 TCP처럼 연결 관리·재전송·순서 보장을 기본 제공하지 않습니다.
- 시험에서는 “오류 검출”과 “오류 복구/재전송”을 구분하는 문제가 자주 나옵니다.

- 체크섬: 전송 오류 검출용, 자동 오류 수정 기능은 아님.
- TCP 신뢰성: 순서 번호 + ACK + 재전송 + 체크섬이 함께 작동.
- TCP vs UDP: TCP는 신뢰성 제어가 강하고, UDP는 가볍지만 보장 기능이 적음.

오늘 주제: SQL/DB - OUTER JOIN의 ON 조건과 WHERE 조건 차이

1. 핵심 요약
- LEFT OUTER JOIN은 왼쪽 테이블의 행을 모두 보존하는 조인입니다.
- 오른쪽 테이블에 매칭되는 행이 없으면 오른쪽 컬럼들은 NULL로 채워집니다.
- ON절은 “어떤 행끼리 조인할지”를 정하는 조건입니다.
- WHERE절은 조인이 끝난 뒤 “최종 결과에서 어떤 행을 남길지”를 거르는 조건입니다.
- 그래서 LEFT JOIN 후 WHERE절에 오른쪽 테이블 컬럼 조건을 쓰면 NULL 확장된 행이 제거될 수 있습니다.
- 예를 들어 `WHERE B.status = 'Y'`는 B가 없는 행의 NULL을 TRUE로 보지 않기 때문에 결과에서 빠질 수 있습니다.
- 이 경우 의도와 달리 INNER JOIN처럼 결과가 줄어드는 효과가 생깁니다.
- 왼쪽 행은 보존하면서 오른쪽 테이블의 매칭 조건만 제한하려면 조건을 ON절에 두는 방식을 자주 고려합니다.
- SQLD에서도 OUTER JOIN, NULL 처리, ON/WHERE 조건 위치 차이는 자주 보는 포인트입니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: LEFT OUTER JOIN은 왼쪽 행 보존, 매칭 실패 시 오른쪽 컬럼 NULL.
- SQLD 포인트: ON절은 조인 조건, WHERE절은 조인 후 필터링이라는 처리 의미를 구분.
- 함정 포인트: 오른쪽 테이블 조건을 WHERE에 두면 OUTER JOIN 결과가 INNER JOIN처럼 줄어들 수 있음.

3. OX 문제
LEFT OUTER JOIN에서 오른쪽 테이블의 컬럼 조건을 WHERE절에 두면, 조인 후 NULL로 확장된 행이 조건에서 걸러져 결과가 INNER JOIN처럼 줄어들 수 있다.
