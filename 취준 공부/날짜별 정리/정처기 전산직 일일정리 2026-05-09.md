오늘 주제: 네트워크 - IPv4 TTL과 패킷 생존 시간

1. 핵심 요약
- TTL은 Time To Live의 약자로, IPv4 패킷이 네트워크 안에서 무한히 떠도는 것을 막기 위한 필드입니다.
- 이름은 “시간”처럼 보이지만, 실제 시험에서는 보통 라우터를 지나는 홉 수와 연결해 이해하면 됩니다.
- 송신자가 패킷에 TTL 값을 넣어 보내면, 라우터는 패킷을 전달할 때마다 TTL을 1씩 감소시킵니다.
- TTL 값이 0이 되면 라우터는 더 이상 그 패킷을 전달하지 않고 폐기할 수 있습니다.
- 이 구조가 없으면 라우팅 오류나 순환 경로 때문에 패킷이 네트워크를 계속 맴돌 수 있습니다.
- TTL이 만료되면 라우터는 송신자에게 ICMP Time Exceeded 메시지를 보낼 수 있습니다.
- traceroute/tracert 명령은 TTL을 일부러 1, 2, 3처럼 늘려 보내며 경로상의 라우터를 확인하는 원리를 씁니다.
- 즉 TTL은 IP 계층에서 패킷의 생존 범위를 제한하는 안전장치라고 보면 됩니다.

2. 자주 나오는 포인트
- TTL은 라우터를 지날 때마다 감소하고, 0이 되면 패킷 폐기 가능.
- TTL의 핵심 목적은 라우팅 루프에 의한 패킷 무한 순환 방지.
- ICMP Time Exceeded, traceroute 동작 원리와 함께 자주 연결됩니다.

3. OX 문제
IPv4 패킷의 TTL 값은 라우터를 지날 때마다 감소하며, 값이 0이 되면 패킷은 폐기될 수 있다.

오늘 주제: 네트워크 - HTTP 쿠키와 세션 상태 유지

1. 핵심 요약
- HTTP는 기본적으로 무상태 프로토콜이라, 서버는 각 요청을 독립적인 요청으로 봅니다.
- 쿠키는 서버가 클라이언트 브라우저에 저장하도록 보내는 작은 데이터입니다.
- 서버는 응답에 Set-Cookie 헤더를 넣고, 브라우저는 이후 요청에 Cookie 헤더로 값을 다시 보냅니다.
- 이 구조 덕분에 로그인 상태, 장바구니 식별자, 사용자 설정 같은 상태 유지가 가능합니다.
- 세션은 실제 사용자 상태 정보를 서버 쪽에 저장하고, 클라이언트에는 세션 ID만 쿠키로 보내는 방식이 흔합니다.
- 쿠키에 비밀번호나 주민번호 같은 민감 정보를 그대로 저장하면 탈취 시 위험합니다.
- Secure는 HTTPS에서만 전송, HttpOnly는 JavaScript 접근 제한, SameSite는 CSRF 위험 완화에 도움을 줍니다.
- 시험에서는 “HTTP는 무상태 → 쿠키/세션으로 상태 유지” 흐름을 이해하는 것이 중요합니다.

2. 자주 나오는 포인트
- 쿠키: 클라이언트 저장 / 세션: 서버 저장 + 세션 ID 전달 구조로 자주 비교됩니다.
- Set-Cookie는 서버 응답, Cookie는 클라이언트 요청 헤더입니다.
- 쿠키 보안 속성 Secure, HttpOnly, SameSite의 역할을 구분하세요.

- 단, 참여도는 “반드시 관계가 있어야 하는가”이고, 관계 차수는 “몇 개와 연결될 수 있는가”이므로 구분해야 합니다.

오늘 주제: 네트워크 - 이더넷 프레임과 MAC 주소

1. 핵심 요약
- MAC 주소는 데이터 링크 계층, 즉 2계층에서 사용하는 장비 식별 주소입니다.
- 이더넷 LAN 안에서 실제 프레임을 어느 장비로 보낼지 판단할 때 목적지 MAC 주소를 사용합니다.
- IP 주소가 최종 목적지를 나타내는 논리 주소라면, MAC 주소는 현재 네트워크 구간에서의 전달 주소에 가깝습니다.
- 같은 서브넷 통신에서는 ARP를 이용해 목적지 IP에 대응하는 MAC 주소를 알아낸 뒤 직접 프레임을 보냅니다.
- 다른 네트워크로 보낼 때는 목적지 IP는 최종 서버 IP로 두지만, 목적지 MAC은 기본 게이트웨이의 MAC 주소가 됩니다.
- 라우터는 들어온 프레임을 벗겨 IP 패킷을 확인한 뒤, 다음 네트워크 구간에 맞는 새 프레임을 만들어 보냅니다.

- 하나의 광섬유에 여러 파장의 신호를 동시에 실어 보낼 수 있어 전송 용량을 크게 늘릴 수 있습니다.
- 전기 통신의 FDM이 주파수 대역을 나누는 방식이라면, WDM은 광통신에서 빛의 파장을 나누는 방식입니다.
- 예를 들어 빨간빛, 파란빛처럼 서로 다른 파장의 빛을 같은 광섬유에 함께 보내고, 수신 측에서 다시 분리한다고 이해하면 쉽습니다.
- 광섬유 인프라를 새로 많이 깔지 않고도 회선 용량을 확장할 수 있다는 장점이 있습니다.
- 정보통신/NCS에서는 다중화 방식 비교 문제가 자주 나오므로 TDM, FDM, WDM을 함께 구분해야 합니다.

2. 자주 나오는 포인트
- WDM: 광통신, 파장 분할, 여러 광 신호 동시 전송
- FDM: 주파수 대역 분할 / TDM: 시간 슬롯 분할
- WDM은 광섬유의 전송 용량 확대와 관련된 기술로 자주 출제됩니다.

3. OX 문제
