오늘 주제: 운영체제 - Clock 페이지 교체 알고리즘

1. 핵심 요약
- 페이지 교체 알고리즘은 메모리에 빈 프레임이 없을 때 어떤 페이지를 내보낼지 정하는 방식입니다.
- FIFO는 먼저 들어온 페이지를 내보내고, LRU는 가장 오래 사용되지 않은 페이지를 내보냅니다.
- 하지만 LRU를 정확히 구현하려면 최근 사용 시점을 계속 관리해야 해서 비용이 큽니다.
- Clock 알고리즘은 LRU를 간단히 흉내 내는 근사 알고리즘입니다.
- 각 페이지에 참조 비트를 두고, 시계 바늘처럼 프레임을 순서대로 검사합니다.
- 참조 비트가 1이면 최근 사용된 페이지로 보고 0으로 바꾼 뒤 한 번 더 기회를 줍니다.
- 참조 비트가 0이면 최근에 사용되지 않은 것으로 판단해 교체 대상으로 선택합니다.
- 그래서 Clock 알고리즘은 Second Chance 알고리즘이라고도 불립니다.

2. 자주 나오는 포인트
- Clock = 참조 비트 사용 + 시계 바늘 방식 + Second Chance.
- 참조 비트 1은 바로 교체하지 않고 0으로 바꾼 뒤 한 번 더 기회를 준다.
- LRU를 정확히 구현한 것은 아니고, 최근 사용 여부를 간단히 반영하는 근사 방식이다.

3. OX 문제
Clock 페이지 교체 알고리즘은 참조 비트를 이용해 최근에 사용된 페이지에는 한 번 더 기회를 주고, 참조 비트가 0인 페이지를 교체 대상으로 선택한다.

오늘 주제: SQL/DB - DCL WITH GRANT OPTION과 권한 위임

1. 핵심 요약
- DCL은 데이터베이스 권한을 제어하는 명령어입니다.
- 대표 명령어는 GRANT와 REVOKE입니다.
- GRANT는 사용자에게 SELECT, INSERT, UPDATE 같은 권한을 부여합니다.
- REVOKE는 이미 부여된 권한을 회수합니다.
- WITH GRANT OPTION은 권한을 받은 사용자가 그 권한을 다른 사용자에게 다시 부여할 수 있게 하는 옵션입니다.
- 예를 들어 A가 B에게 SELECT 권한을 WITH GRANT OPTION으로 주면, B도 C에게 SELECT 권한을 줄 수 있습니다.
- 동작 원리는 권한 자체뿐 아니라 “권한을 위임할 수 있는 권한”까지 함께 주는 것입니다.
- SQLD에서도 DML/DDL/DCL/TCL 분류와 GRANT·REVOKE 역할은 자주 확인하는 기본 포인트입니다.
- 단, 권한 위임이 많아지면 보안 관리가 복잡해지므로 누가 어떤 권한을 갖는지 추적이 중요합니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: GRANT는 권한 부여, REVOKE는 권한 회수, 둘 다 DCL입니다.
- WITH GRANT OPTION은 “받은 권한을 다른 사용자에게 다시 부여 가능”이 핵심입니다.
- 권한 제어는 데이터 조회·수정 명령이 아니라 접근 통제와 보안 관리 관점에서 봐야 합니다.

3. OX 문제
SQL에서 권한을 부여받은 사용자가 WITH GRANT OPTION으로 권한을 받으면, 그 권한을 다른 사용자에게 다시 부여할 수 있다.

오늘 주제: 운영체제 - 디스크 스케줄링 SCAN 알고리즘

1. 핵심 요약
- 디스크 스케줄링은 여러 입출력 요청이 있을 때 디스크 헤드를 어떤 순서로 이동시킬지 정하는 방법입니다.
- 목표는 헤드 이동 거리를 줄이고, 평균 대기 시간을 낮추며, 특정 요청이 너무 오래 기다리지 않게 하는 것입니다.
- FCFS는 요청이 들어온 순서대로 처리해 단순하지만, 헤드 이동이 비효율적일 수 있습니다.
- SSTF는 현재 헤드 위치에서 가장 가까운 요청을 먼저 처리해 이동 거리를 줄이지만, 먼 요청이 계속 밀리는 기아 현상이 생길 수 있습니다.
- SCAN은 엘리베이터처럼 한 방향으로 이동하면서 요청을 처리하고, 끝 방향에 도달하면 반대 방향으로 이동합니다.
- 그래서 가까운 요청만 계속 고르는 SSTF보다 기아 문제를 완화할 수 있습니다.
- C-SCAN은 한 방향으로만 요청을 처리하고, 끝에 도달하면 반대 방향 처리 없이 처음 쪽으로 돌아가 다시 같은 방향으로 처리합니다.
- 시험에서는 FCFS, SSTF, SCAN, C-SCAN의 처리 기준 차이를 비교하는 문제가 자주 나옵니다.

2. 자주 나오는 포인트
- FCFS: 요청 순서대로 처리, 단순하지만 헤드 이동 비효율 가능
- SSTF: 가장 가까운 요청 우선, 평균 이동 거리는 줄지만 기아 가능
- SCAN: 한 방향 이동 후 반대 방향으로 처리, 엘리베이터 알고리즘

3. OX 문제
