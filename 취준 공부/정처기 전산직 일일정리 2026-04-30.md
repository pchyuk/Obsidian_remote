오늘 주제: 네트워크 - ARP와 MAC 주소 해석

1. 핵심 요약
- ARP(Address Resolution Protocol)는 IP 주소를 MAC 주소로 알아내기 위한 프로토콜입니다.
- IP 주소는 논리적 주소이고, MAC 주소는 같은 LAN 안에서 실제 프레임 전달에 쓰이는 물리 주소입니다.
- 같은 네트워크 안에서 통신하려면 최종적으로 목적지 장비의 MAC 주소를 알아야 합니다.
- 송신자는 목적지 IP는 알지만 MAC 주소를 모를 때 ARP Request를 보냅니다.
- ARP 요청은 “이 IP 주소 가진 장비 누구야?”라는 형태로 브로드캐스트됩니다.
- 해당 IP를 가진 장비는 자신의 MAC 주소를 담아 ARP Reply를 보냅니다.
- 일반적으로 ARP 응답은 요청한 장비에게 유니캐스트로 돌아옵니다.
- 알아낸 IP-MAC 대응 정보는 ARP 캐시에 잠시 저장되어 매번 요청하지 않게 합니다.
- 시험에서는 ARP는 IP→MAC, RARP는 MAC→IP 방향이라는 구분이 자주 나옵니다.

2. 자주 나오는 포인트
- ARP: IPv4에서 IP 주소를 MAC 주소로 변환
- ARP Request: 브로드캐스트 / ARP Reply: 보통 유니캐스트
- 같은 LAN 내부 통신에서는 IP만으로 부족하고, 실제 프레임 전달에는 MAC 주소가 필요함

3. OX 문제
ARP는 같은 네트워크에서 목적지 IP 주소에 대응하는 MAC 주소를 알아내기 위해 사용되며, 일반적으로 ARP 요청은 브로드캐스트되고 ARP 응답은 유니캐스트로 돌아온다.  
O / X?

오늘 주제: SQL/DB - 트랜잭션 격리 수준과 동시성 이상 현상

1. 핵심 요약
- 트랜잭션은 데이터베이스에서 하나의 논리적 작업 단위입니다.
- 예를 들어 계좌 이체는 출금과 입금이 모두 성공해야 하며, 하나만 성공하면 안 됩니다.
- 격리 수준은 여러 트랜잭션이 동시에 실행될 때 서로의 작업 결과를 얼마나 분리해서 볼 것인지 정하는 기준입니다.
- 격리 수준이 낮으면 여러 작업을 동시에 처리하기 쉬워 성능과 동시성이 좋아질 수 있습니다.
- 대신 아직 확정되지 않은 데이터를 읽는 Dirty Read 같은 문제가 생길 수 있습니다.
- 격리 수준이 높으면 데이터 일관성은 좋아지지만, 잠금이나 대기 시간이 늘어 성능이 떨어질 수 있습니다.
- 대표적인 동시성 이상 현상에는 Dirty Read, Non-repeatable Read, Phantom Read가 있습니다.
- SQLD에서도 자주 보는 포인트는 트랜잭션의 ACID, COMMIT/ROLLBACK, 격리 수준과 이상 현상의 관계입니다.

2. 자주 나오는 포인트
- Dirty Read: 다른 트랜잭션이 아직 COMMIT하지 않은 데이터를 읽는 현상입니다.
- Non-repeatable Read: 같은 행을 다시 읽었는데 중간에 다른 트랜잭션이 수정/커밋해서 값이 달라지는 현상입니다.
- Phantom Read: 같은 조건으로 다시 조회했을 때, 다른 트랜잭션의 INSERT/DELETE 때문에 결과 행 집합이 달라지는 현상입니다.

3. OX 문제
트랜잭션 격리 수준이 낮아질수록 일반적으로 동시성은 높아질 수 있지만 Dirty Read 같은 이상 현상이 발생할 가능성도 커진다.
O / X?

오늘 주제: 자료구조 - 선택 정렬과 시간복잡도

1. 핵심 요약
- 선택 정렬은 정렬되지 않은 구간에서 가장 작은 값 또는 가장 큰 값을 선택해 앞쪽부터 차례로 배치하는 정렬 방식입니다.
- 예를 들어 오름차순 정렬이라면, 첫 번째 회차에는 전체에서 최솟값을 찾아 맨 앞 원소와 교환합니다.
- 두 번째 회차에는 두 번째 위치부터 끝까지 중 최솟값을 찾아 두 번째 위치에 놓습니다.
- 이렇게 정렬된 구간은 앞에서부터 하나씩 늘어나고, 정렬되지 않은 구간은 줄어듭니다.
- 핵심은 “비교는 많이 하지만, 교환은 비교적 적다”는 점입니다.
- 선택 정렬은 입력이 이미 정렬되어 있어도 최솟값을 찾기 위해 계속 비교해야 하므로 최선·평균·최악 시간복잡도가 모두 O(n²)입니다.
- 구현은 단순하지만 데이터가 많을수록 비효율적이라 실무보다는 시험용 기본 정렬 개념으로 자주 나옵니다.

2. 자주 나오는 포인트
- 선택 정렬: 남은 원소 중 최솟값/최댓값을 선택해 제자리 교환
- 시간복잡도: 최선·평균·최악 모두 O(n²)
- 버블 정렬·삽입 정렬과 함께 단순 정렬 계열로 비교 출제 자주 나옴

오늘 주제: SQL/DB - 집합 연산 UNION과 UNION ALL

1. 핵심 요약
- 집합 연산은 여러 SELECT 결과를 하나의 결과처럼 합치는 SQL 기능입니다.
- 대표적으로 UNION, UNION ALL, INTERSECT, MINUS 또는 EXCEPT가 있습니다.
- UNION은 두 SELECT 결과를 합치되, 중복 행을 제거합니다.
- UNION ALL은 중복 제거 없이 결과를 그대로 이어 붙입니다.
- 그래서 UNION ALL은 중복 제거 정렬·비교 비용이 없어 보통 더 빠를 수 있습니다.
- 단, 결과에 중복이 나오면 안 되는 문제라면 UNION을 써야 합니다.
- 집합 연산을 쓰려면 각 SELECT문의 컬럼 개수가 같아야 합니다.
- 대응되는 컬럼의 데이터 타입도 서로 호환되어야 합니다.
- SQLD에서도 UNION과 UNION ALL의 중복 처리 차이, 집합 연산의 컬럼 조건이 자주 나옵니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: UNION은 중복 제거, UNION ALL은 중복 포함.
- 집합 연산 조건: SELECT 컬럼 개수와 대응 컬럼 타입이 맞아야 함.
- 성능 포인트: 중복 제거가 필요 없으면 UNION ALL이 더 효율적일 수 있음.

3. OX 문제
SQL 집합 연산에서 UNION은 중복 행을 제거하지만, UNION ALL은 중복 행까지 그대로 포함한다.

오늘 주제: 네트워크 - NAT와 사설 IP

1. 핵심 요약
- NAT는 Network Address Translation, 즉 네트워크 주소 변환 기술입니다.
- 사설 IP는 내부 LAN에서만 쓰는 주소이고, 인터넷에서는 직접 라우팅되지 않습니다.
- 그래서 내부 PC나 스마트폰이 인터넷에 나가려면 보통 공유기/방화벽이 사설 IP를 공인 IP로 바꿔 줍니다.
- 동작 원리는 “내부 출발지 IP:포트 → 공인 IP:임시 포트” 형태로 매핑 테이블을 만들고, 응답이 오면 다시 원래 내부 장비로 돌려보내는 방식입니다.
- 이때 포트까지 함께 변환하는 방식을 PAT 또는 NAPT라고 하며, 가정용 공유기에서 흔히 사용됩니다.
- NAT는 IPv4 공인 주소 부족 문제를 완화하는 데 큰 역할을 했습니다.
- 다만 NAT는 주소 변환 기술이지, 그 자체가 완전한 보안 기술은 아닙니다.
- 시험에서는 사설 IP, 공인 IP, 주소 변환, 포트 변환, IPv4 주소 절약 효과를 함께 묶어서 자주 봅니다.

2. 자주 나오는 포인트
- NAT: 사설 IP ↔ 공인 IP 주소 변환
- PAT/NAPT: 여러 내부 장비가 하나의 공인 IP를 포트 번호로 구분해 사용
- 사설 IP 대역 예: 10.0.0.0/8, 172.16.0.0~172.31.255.255, 192.168.0.0/16

3. OX 문제
NAT는 사설 IP 주소를 사용하는 내부 호스트가 공인 IP 주소로 외부 네트워크와 통신할 수 있도록 주소를 변환하는 기술이다.

1. 핵심 요약

오늘 주제: 네트워크 - DHCP와 동적 IP 할당

1. 핵심 요약
- DHCP는 Dynamic Host Configuration Protocol의 약자입니다.
- 클라이언트에게 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 주소 등을 자동으로 할당합니다.
- 핵심은 “자동 설정”과 “임대 방식”입니다.

- 대표 흐름은 DORA: Discover → Offer → Request → ACK입니다.
- 처음 접속한 클라이언트는 IP가 없으므로 브로드캐스트로 DHCP 서버를 찾습니다.

- 일반적으로 서버는 UDP 67번, 클라이언트는 UDP 68번 포트를 사용합니다.
- DNS는 “이름을 IP로 변환”, DHCP는 “IP 설정 정보를 자동 배정”이라는 차이를 꼭 구분하세요.

2. 자주 나오는 포인트
- DHCP = IP 자동 할당 + 임대/갱신 개념
- DORA 순서: Discover → Offer → Request → ACK
- DHCP 서버 UDP 67번, 클라이언트 UDP 68번
