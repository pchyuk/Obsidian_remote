찬혁 님, 오늘은 **운영체제 - CPU 스케줄링**입니다.

CPU 스케줄링은 여러 프로세스 중 어떤 프로세스에게 CPU를 배정할지 결정하는 운영체제의 핵심 기능입니다.  
스케줄링 방식은 크게 **선점형**과 **비선점형**으로 나뉩니다.  
비선점형은 한 프로세스가 CPU를 잡으면 종료되거나 대기 상태가 될 때까지 CPU를 빼앗지 않습니다.  
반면 선점형은 타이머 인터럽트나 더 높은 우선순위 프로세스 등장 시 CPU를 강제로 회수할 수 있습니다.  
대표적인 비선점형 방식에는 FCFS, SJF, HRN이 있고, 선점형 방식에는 Round Robin, SRT, 선점 우선순위 스케줄링이 있습니다.  
공기업 전산직 필기에서는 각 알고리즘의 특징과 평균 대기시간 계산 문제가 자주 나옵니다.

자주 나오는 포인트:
- **Round Robin**은 시간 할당량이 핵심이며, 너무 작으면 문맥 교환 오버헤드가 커집니다.
- **SJF/SRT**는 실행 시간이 짧은 작업을 우선해 평균 대기시간을 줄이는 데 유리합니다.
- **FCFS**는 구현은 단순하지만 convoy effect가 발생할 수 있습니다.

OX 문제:  
**운영체제에서 선점형 스케줄링은 실행 중인 프로세스의 CPU를 강제로 회수해 다른 프로세스에 배정할 수 있다.**

오늘의 주제: SQL/DB - 정규화

정규화는 데이터베이스에서 중복을 줄이고 이상 현상을 방지하기 위해 테이블을 구조적으로 분해하는 과정입니다.  
이상 현상에는 삽입 이상, 삭제 이상, 갱신 이상이 있습니다.  
제1정규형은 속성값이 원자값이어야 한다는 조건입니다.  
제2정규형은 부분 함수 종속을 제거하는 단계로, 주로 복합키일 때 중요합니다.  
제3정규형은 이행적 함수 종속을 제거하는 단계입니다.  
즉, 기본키가 아닌 속성이 또 다른 기본키가 아닌 속성을 결정하면 분해 대상이 됩니다.  
정규화를 과도하게 하면 조인이 많아져 성능이 떨어질 수 있어, 실무에서는 반정규화를 고려하기도 합니다.

자주 나오는 포인트:
- 1NF: 반복 속성 제거, 원자값
- 2NF: 부분 함수 종속 제거
- 3NF: 이행적 함수 종속 제거

OX 문제:  
관계형 데이터베이스의 제3정규형(3NF)은 이행적 함수 종속을 제거하는 것을 목표로 한다.

오늘의 주제: 네트워크 - TCP와 UDP

TCP와 UDP는 전송 계층에서 사용하는 대표적인 프로토콜입니다.  
TCP는 연결형 프로토콜로, 데이터를 보내기 전에 연결 설정 과정을 거칩니다.  
대표적으로 3-way handshake를 통해 송신자와 수신자가 통신 준비 상태인지 확인합니다.  
TCP는 데이터의 순서 보장, 오류 검출, 재전송, 흐름 제어, 혼잡 제어를 제공해 신뢰성이 높습니다.  
반면 UDP는 비연결형 프로토콜로, 연결 설정 없이 데이터그램을 바로 전송합니다.  
그래서 TCP보다 오버헤드가 작고 빠르지만, 데이터 손실이나 순서 뒤바뀜을 자체적으로 보장하지 않습니다.  
UDP는 DNS, VoIP, 실시간 스트리밍처럼 속도와 지연 시간이 중요한 서비스에서 자주 사용됩니다.

자주 나오는 포인트

- TCP: 연결형, 신뢰성 높음, 순서 보장, 재전송 가능
- UDP: 비연결형, 빠름, 오버헤드 작음, 신뢰성 보장 약함
- 3-way handshake는 TCP 연결 설정 과정

OX 문제

UDP는 연결 설정 과정이 없고, 데이터그램 단위로 전송하며, TCP보다 전송 신뢰성을 항상 더 높게 보장한다.
