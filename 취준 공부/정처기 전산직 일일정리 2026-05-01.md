오늘 주제: SQL/DB - DCL과 권한 제어 GRANT/REVOKE

1. 핵심 요약
- DCL은 Data Control Language, 즉 데이터 제어어입니다.
- 데이터 자체를 조회하거나 수정하는 명령이 아니라, 사용자 권한을 제어하는 SQL 명령어입니다.
- 대표 명령어는 GRANT와 REVOKE입니다.
- GRANT는 사용자나 역할에게 SELECT, INSERT, UPDATE, DELETE 같은 권한을 부여합니다.
- REVOKE는 이미 부여된 권한을 회수합니다.
- 예를 들어 특정 사용자에게 사원 테이블 조회 권한만 주면, 그 사용자는 데이터를 볼 수는 있지만 수정은 못 하게 제한할 수 있습니다.
- SQLD에서도 DML·DDL·DCL·TCL 분류는 자주 나옵니다.
- DML은 데이터 조작, DDL은 객체 정의, DCL은 권한 제어, TCL은 트랜잭션 제어로 구분하면 됩니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: GRANT는 권한 부여, REVOKE는 권한 회수입니다.
- DCL은 테이블 생성·삭제가 아니라 접근 권한 관리와 관련됩니다.
- DML: SELECT/INSERT/UPDATE/DELETE, DDL: CREATE/ALTER/DROP, TCL: COMMIT/ROLLBACK과 구분하세요.

3. OX 문제
DCL은 데이터베이스 객체에 대한 권한을 부여하거나 회수하는 명령어로, 대표적으로 GRANT와 REVOKE가 있다.

오늘 주제: 네트워크 - 라우팅 프로토콜 RIP와 OSPF

1. 핵심 요약
- 라우팅 프로토콜은 라우터들이 목적지까지의 경로 정보를 서로 교환하고 최적 경로를 선택하는 규칙입니다.
- RIP는 거리 벡터 방식 라우팅 프로토콜입니다.
- 거리 벡터 방식은 “목적지까지 얼마나 떨어져 있는가”를 기준으로 경로를 판단합니다.
- RIP는 주로 홉 수, 즉 거쳐 가는 라우터 개수를 기준으로 최적 경로를 고릅니다.
- 구조가 단순하지만 최대 홉 수 제한이 있어 큰 네트워크에는 적합하지 않습니다.
- OSPF는 링크 상태 방식 라우팅 프로토콜입니다.
- 링크 상태 방식은 각 라우터가 네트워크 연결 상태 정보를 모아 전체 구조를 파악한 뒤 최단 경로를 계산합니다.
- OSPF는 SPF 알고리즘을 사용해 목적지까지의 최적 경로를 계산합니다.
- 시험에서는 RIP는 단순하지만 규모 한계가 있고, OSPF는 복잡하지만 대규모 네트워크에 더 적합하다고 이해하면 됩니다.

2. 자주 나오는 포인트
- RIP: 거리 벡터, 홉 수 기준, 소규모 네트워크에 적합
- OSPF: 링크 상태, SPF 알고리즘, 대규모 네트워크에 적합
- 거리 벡터와 링크 상태 라우팅의 차이를 묻는 문제가 자주 나옵니다.

3. OX 문제

오늘 주제: SQL/DB - DDL과 DELETE·TRUNCATE·DROP 구분

1. 핵심 요약
- DDL은 Data Definition Language, 즉 데이터베이스 구조를 정의·변경·삭제하는 명령어입니다.
- 대표 명령어는 CREATE, ALTER, DROP, TRUNCATE입니다.
- CREATE는 테이블·뷰·인덱스 같은 객체를 만들고, ALTER는 이미 만든 객체의 구조를 변경합니다.
- DROP은 테이블 같은 객체 자체를 삭제합니다. 구조와 데이터가 함께 사라진다고 이해하면 됩니다.
- DELETE는 헷갈리기 쉽지만 DDL이 아니라 DML입니다. 테이블 구조는 유지하고 행 데이터만 삭제합니다.
- TRUNCATE는 테이블 구조는 남기고 전체 데이터를 빠르게 삭제하는 DDL 성격의 명령으로 자주 출제됩니다.
- SQLD에서도 DELETE·TRUNCATE·DROP 구분은 단골입니다. “행 삭제냐, 전체 데이터 초기화냐, 객체 삭제냐”로 나누면 좋습니다.
- 실무 감각으로 보면 DELETE는 조건 삭제, TRUNCATE는 통째 비우기, DROP은 테이블 자체 제거입니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: DDL은 CREATE, ALTER, DROP, TRUNCATE / DML은 SELECT, INSERT, UPDATE, DELETE.
- DELETE는 WHERE절로 일부 행 삭제 가능, DROP은 객체 자체 삭제라 테이블 구조도 사라짐.
- TRUNCATE는 전체 행 삭제에 가깝지만 DDL로 분류되는 경우가 많아 DELETE와 구분해야 함.

3. OX 문제
DDL의 DROP은 테이블 구조 자체를 제거하는 명령이고, DELETE는 테이블 구조는 유지한 채 행 데이터를 삭제하는 DML 명령이다.

오늘 주제: 네트워크 - IPv6 주소 체계와 특징

1. 핵심 요약
- IPv6는 IPv4 주소 부족 문제를 해결하기 위해 도입된 128비트 IP 주소 체계입니다.
- IPv4는 32비트라 주소 개수가 제한적이지만, IPv6는 훨씬 넓은 주소 공간을 제공합니다.
- IPv6 주소는 보통 16비트씩 8개 그룹을 콜론(:)으로 구분해 16진수로 표현합니다.
- 예: 2001:0db8:0000:0000:0000:ff00:0042:8329 같은 형태입니다.
- 연속된 0 그룹은 한 번만 ::로 축약할 수 있어 주소를 짧게 쓸 수 있습니다.
- IPv6는 브로드캐스트를 사용하지 않고, 멀티캐스트와 애니캐스트를 활용합니다.
- 주소 자동 설정 기능을 지원해 장비가 네트워크에 접속할 때 주소 설정을 더 쉽게 할 수 있습니다.
- 시험에서는 “IPv4는 32비트, IPv6는 128비트” 구분이 가장 먼저 나옵니다.

2. 자주 나오는 포인트
- IPv6 주소 길이: 128비트, IPv4는 32비트
- IPv6 표기: 콜론(:)으로 구분한 16진수 8그룹
- IPv6 특징: 주소 공간 확대, 브로드캐스트 미사용, 자동 설정 지원

3. OX 문제
IPv6는 128비트 주소 체계를 사용하며, 주소 공간이 IPv4보다 훨씬 커서 IPv4 주소 부족 문제를 완화하기 위해 도입되었다.

오늘 주제: SQL/DB - WHERE 조건과 LIKE 패턴 검색

1. 핵심 요약
- WHERE절은 테이블에서 조건에 맞는 행만 걸러낼 때 사용합니다.
- SELECT가 “어떤 컬럼을 볼지” 정한다면, WHERE는 “어떤 행을 볼지” 정하는 역할입니다.
- LIKE는 문자열이 특정 패턴과 일치하는지 확인할 때 쓰는 조건 연산자입니다.
- `%`는 0개 이상의 임의 문자를 의미합니다. 예를 들어 `LIKE '김%'`는 김으로 시작하는 값을 찾습니다.
- `_`는 정확히 1개의 임의 문자를 의미합니다. 예를 들어 `LIKE 'A_'`는 A로 시작하는 두 글자 값을 찾습니다.
- `BETWEEN A AND B`는 범위 조건, `IN (...)`은 여러 후보 값 중 포함 여부를 검사합니다.
- NULL은 일반 비교 연산자인 `= NULL`로 찾지 않고, 반드시 `IS NULL` 또는 `IS NOT NULL`을 사용합니다.
- SQLD에서도 WHERE, LIKE, IN, BETWEEN, IS NULL은 SELECT 기본 문법에서 자주 나오는 포인트입니다.

2. 자주 나오는 포인트
- 정처기/SQLD 공통: `LIKE 'A%'`는 A로 시작, `LIKE '%A'`는 A로 끝, `LIKE '%A%'`는 A 포함입니다.
- SQLD 포인트: `%`는 0개 이상, `_`는 정확히 1개 문자라는 차이를 구분해야 합니다.
- 주의 포인트: NULL 비교는 `= NULL`이 아니라 `IS NULL`입니다.

3. OX 문제
SQL의 WHERE절에서 LIKE는 문자열 패턴 검색에 사용되며, 와일드카드 %는 0개 이상의 임의 문자, _는 정확히 1개의 임의 문자를 의미한다.
