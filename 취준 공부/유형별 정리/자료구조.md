### 2026-04-27 09:01 - 자료구조 - 스택과 큐

[정보처리기사/전산직 필기]
오늘 주제: 자료구조 - 스택과 큐

1. 핵심 요약
- 스택(Stack)은 데이터를 한쪽 끝에서만 넣고 빼는 선형 자료구조입니다.
- 가장 나중에 들어온 데이터가 가장 먼저 나가므로 후입선출, 즉 LIFO 구조라고 합니다.
- 접시를 쌓았다가 위에서부터 꺼내는 모습을 떠올리면 이해하기 쉽습니다.
- 큐(Queue)는 한쪽에서는 삽입, 반대쪽에서는 삭제가 일어나는 선형 자료구조입니다.
- 가장 먼저 들어온 데이터가 가장 먼저 나가므로 선입선출, 즉 FIFO 구조입니다.
- 줄 서기처럼 먼저 온 요청을 먼저 처리해야 하는 상황에 잘 맞습니다.
- 스택은 함수 호출, 재귀, 괄호 검사, 뒤로 가기 기능에 자주 쓰입니다.
- 큐는 작업 대기열, 프린터 출력 순서, 버퍼, BFS 탐색에서 자주 활용됩니다.

2. 자주 나오는 포인트
- 스택: LIFO, push/pop, top 개념을 묻는 문제가 자주 나옵니다.
- 큐: FIFO, enqueue/dequeue, front/rear 개념을 구분해야 합니다.
- BFS는 큐, DFS는 스택 또는 재귀와 연결해서 기억하면 좋습니다.

3. OX 문제
스택은 후입선출(LIFO) 구조라서 재귀 호출, 함수 호출 복귀 주소 관리, 괄호 검사에 활용되고, 큐는 선입선출(FIFO) 구조라서 작업 대기열이나 BFS 탐색에 활용된다.

오늘 주제: 자료구조 - 이진 트리와 순회

1. 핵심 요약
- 트리는 계층 구조를 표현하는 비선형 자료구조입니다.
- 이진 트리는 각 노드가 최대 2개의 자식 노드를 가지는 트리입니다.
- 자식은 보통 왼쪽 자식, 오른쪽 자식으로 구분합니다.
- 트리 순회는 모든 노드를 빠짐없이 한 번씩 방문하는 방법입니다.
- 전위 순회는 루트 → 왼쪽 → 오른쪽 순서입니다.
- 중위 순회는 왼쪽 → 루트 → 오른쪽 순서입니다.
- 후위 순회는 왼쪽 → 오른쪽 → 루트 순서입니다.
- 이름에서 “전/중/후”는 루트 노드를 언제 방문하는지를 기준으로 보면 쉽습니다.

2. 자주 나오는 포인트
- 전위 순회: 루트가 가장 먼저 나온다.
- 중위 순회: 이진 탐색 트리에서 사용하면 값이 오름차순으로 나올 수 있다.
- 후위 순회: 하위 노드를 먼저 처리하므로 수식 트리 계산, 디렉터리 삭제 등에 응용된다.

3. OX 문제
이진 트리의 중위 순회는 루트 노드를 먼저 방문한 뒤 왼쪽 서브트리와 오른쪽 서브트리를 차례로 방문하는 방식이다.

### 2026-04-29 16:00 - 자료구조 - 그래프 탐색 DFS와 BFS

[정보처리기사/전산직 필기]
오늘 주제: 자료구조 - 그래프 탐색 DFS와 BFS

1. 핵심 요약
- 그래프는 정점과 간선으로 구성된 자료구조입니다.
- DFS는 깊이 우선 탐색으로, 한 방향으로 갈 수 있을 만큼 깊게 들어간 뒤 되돌아옵니다.
- DFS는 보통 스택이나 재귀 호출로 구현합니다.
- BFS는 너비 우선 탐색으로, 시작 정점에서 가까운 정점부터 차례로 방문합니다.
- BFS는 먼저 들어온 정점을 먼저 처리해야 하므로 큐를 사용합니다.
- 무가중치 그래프에서 BFS는 시작점으로부터 간선 수가 가장 적은 경로를 먼저 찾을 수 있어 최단 경로 문제에 자주 쓰입니다.
- DFS는 경로 존재 여부, 사이클 탐지, 백트래킹 문제와 자주 연결됩니다.
- 시험에서는 “DFS=스택/재귀”, “BFS=큐”, “BFS=무가중치 최단 경로”를 연결해서 기억하면 좋습니다.

2. 자주 나오는 포인트
- DFS: 스택 또는 재귀, 깊게 탐색, 백트래킹과 관련
- BFS: 큐, 가까운 정점부터 탐색, 무가중치 최단 경로에 활용
- 방문 체크를 하지 않으면 사이클이 있는 그래프에서 무한 반복될 수 있음

3. OX 문제
그래프 탐색에서 BFS는 큐를 이용해 시작 정점에서 가까운 정점부터 방문하므로, 간선 가중치가 없는 그래프에서 최단 경로 탐색에 활용될 수 있다.

### 2026-04-30 12:00 - 자료구조 - 선택 정렬과 시간복잡도

[정보처리기사/전산직 필기]
오늘 주제: 자료구조 - 선택 정렬과 시간복잡도

1. 핵심 요약
- 선택 정렬은 정렬되지 않은 구간에서 가장 작은 값 또는 가장 큰 값을 선택해 앞쪽부터 차례로 배치하는 정렬 방식입니다.
- 예를 들어 오름차순 정렬이라면, 첫 번째 회차에는 전체에서 최솟값을 찾아 맨 앞 원소와 교환합니다.
- 두 번째 회차에는 두 번째 위치부터 끝까지 중 최솟값을 찾아 두 번째 위치에 놓습니다.
- 이렇게 정렬된 구간은 앞에서부터 하나씩 늘어나고, 정렬되지 않은 구간은 줄어듭니다.
- 핵심은 “비교는 많이 하지만, 교환은 비교적 적다”는 점입니다.
- 선택 정렬은 입력이 이미 정렬되어 있어도 최솟값을 찾기 위해 계속 비교해야 하므로 최선·평균·최악 시간복잡도가 모두 O(n²)입니다.
- 구현은 단순하지만 데이터가 많을수록 비효율적이라 실무보다는 시험용 기본 정렬 개념으로 자주 나옵니다.

2. 자주 나오는 포인트
- 선택 정렬: 남은 원소 중 최솟값/최댓값을 선택해 제자리 교환
- 시간복잡도: 최선·평균·최악 모두 O(n²)
- 버블 정렬·삽입 정렬과 함께 단순 정렬 계열로 비교 출제 자주 나옴

3. OX 문제
선택 정렬은 매 회차마다 남은 원소 중 최솟값을 찾아 앞쪽 원소와 교환하는 방식이며, 평균 시간복잡도는 O(n log n)이다.

지난 문제 정답

정답: O

해설: CSMA/CD는 전송 중 충돌을 감지하면 전송을 멈추고, 임의의 시간 뒤 다시 전송을 시도하는 방식입니다. 주로 과거 공유 이더넷 환경과 연결됩니다. CSMA/CA는 무선 LAN처럼 충돌 감지가 어려운 환경에서 ACK, 임의 대기, RTS/CTS 등을 이용해 충돌 가능성을 미리 줄이려는 방식입니다. 시험에서는 CD=충돌 감지, CA=충돌 회피로 구분하세요.

오늘 주제: 자료구조 - 퀵 정렬과 피벗 분할

1. 핵심 요약
- 퀵 정렬은 분할 정복 방식의 대표적인 정렬 알고리즘입니다.
- 먼저 기준값인 피벗을 하나 정하고, 피벗보다 작은 값과 큰 값을 나누어 배치합니다.
- 이렇게 나뉜 왼쪽·오른쪽 구간에 다시 같은 과정을 재귀적으로 적용합니다.
- 분할이 균형 있게 이루어지면 전체 정렬 시간은 평균적으로 O(n log n)에 가깝습니다.
- 하지만 피벗이 계속 최솟값이나 최댓값처럼 한쪽으로 치우치면 분할 효과가 떨어집니다.
- 이 경우 한쪽 부분 배열만 거의 n-1 크기로 남아 최악 시간복잡도는 O(n²)이 됩니다.
- 그래서 퀵 정렬은 평균적으로 빠르지만, 피벗 선택에 따라 성능 차이가 커질 수 있습니다.
- 병합 정렬은 보통 안정적으로 O(n log n)이지만 추가 공간이 필요하다는 점과 비교해 자주 출제됩니다.

2. 자주 나오는 포인트
- 퀵 정렬: 피벗 선택 → 분할 → 재귀 정렬 구조
- 평균 시간복잡도는 O(n log n), 최악 시간복잡도는 O(n²)
- 병합 정렬, 선택 정렬, 삽입 정렬과 시간복잡도 비교 문제로 자주 출제

3. OX 문제
퀵 정렬은 피벗을 기준으로 작은 값과 큰 값의 구간을 나누어 정렬하며, 피벗 선택이 계속 한쪽으로 치우치면 최악 시간복잡도가 O(n²)이 될 수 있다.

오늘 주제: 자료구조 - 병합 정렬과 분할 정복

1. 핵심 요약
- 병합 정렬은 분할 정복 방식의 대표적인 정렬 알고리즘입니다.
- 먼저 배열을 계속 반으로 나누어 더 이상 나눌 수 없는 작은 단위로 만듭니다.
- 그다음 작은 배열들을 정렬된 상태로 다시 합치며 전체 배열을 정렬합니다.
- “나누기”는 대략 log n단계, “합치기”는 각 단계마다 전체 원소 n개를 처리합니다.
- 그래서 평균, 최악 시간복잡도가 모두 O(n log n)으로 안정적입니다.
- 퀵 정렬은 평균적으로 빠르지만 피벗이 나쁘면 최악 O(n²)이 될 수 있는데, 병합 정렬은 최악도 O(n log n)입니다.
- 다만 병합 과정에서 보조 배열이 필요할 수 있어 추가 메모리 사용이 단점으로 나옵니다.
- 같은 값의 상대적 순서를 유지하도록 구현할 수 있어 안정 정렬로도 자주 설명됩니다.

2. 자주 나오는 포인트
- 병합 정렬 = 분할 정복, 반으로 나누고 정렬하며 병합
- 시간복잡도: 평균 O(n log n), 최악 O(n log n)
- 퀵 정렬과 비교: 병합 정렬은 안정적이지만 추가 메모리가 필요할 수 있음

3. OX 문제
병합 정렬은 배열을 계속 반으로 나눈 뒤 정렬된 부분 배열을 합치는 분할 정복 방식이며, 일반적으로 최악 시간복잡도도 O(n log n)이다.
