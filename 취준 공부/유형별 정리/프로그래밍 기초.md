### 2026-04-27 11:00 - 프로그래밍 기초 - 시간복잡도와 빅오 표기법

오늘 주제: 프로그래밍 기초 - 시간복잡도와 빅오 표기법

1. 핵심 요약
- 시간복잡도는 입력 크기 n이 커질 때 알고리즘의 수행 시간이 얼마나 빠르게 증가하는지 나타내는 개념입니다.
- 시험에서는 실제 초 단위 시간보다 “연산 횟수의 증가 추세”를 보는 것이 핵심입니다.
- 빅오 표기법 O( )은 보통 최악의 경우를 기준으로 증가 차수를 표현합니다.
- 예를 들어 O(1)은 입력 크기와 거의 무관하게 일정하고, O(n)은 입력이 2배면 연산도 대체로 2배 늘어납니다.
- O(n²)은 이중 반복문처럼 입력이 커질수록 연산량이 제곱 수준으로 증가하는 경우에 자주 나옵니다.
- 빅오에서는 상수 계수와 낮은 차수 항을 생략합니다. 그래서 3n² + 10n + 5도 보통 O(n²)로 봅니다.
- 이유는 n이 매우 커지면 가장 높은 차수의 항이 전체 증가량을 지배하기 때문입니다.
- 공기업 전산직에서는 정렬 알고리즘, 탐색, 반복문 코드 분석과 함께 자주 출제됩니다.

2. 자주 나오는 포인트
- 단일 반복문은 보통 O(n), 이중 반복문은 보통 O(n²)입니다.
- 이진 탐색처럼 탐색 범위를 절반씩 줄이면 O(log n)입니다.
- 빅오는 정확한 실행 시간이 아니라 입력 증가에 따른 “증가 차수”를 나타냅니다.

3. OX 문제
빅오 표기법 O(n²)은 입력 크기 n이 커질 때 수행 시간이 대체로 n의 제곱에 비례해 증가하는 알고리즘의 증가 차수를 나타내며, 실제 실행 시간이 항상 정확히 n²초라는 뜻은 아니다.

- 객체는 데이터인 속성과 동작인 메서드를 함께 가집니다.
- 캡슐화는 관련 있는 데이터와 기능을 하나의 객체 안에 묶는 개념입니다.
- 정보 은닉은 객체 내부 구현을 외부에 직접 드러내지 않고, 필요한 기능만 공개하는 원칙입니다.

- 이렇게 하면 내부 구현이 바뀌어도 외부 코드는 공개된 메서드만 사용하므로 변경 영향이 작아집니다.
- 시험에서는 캡슐화, 정보 은닉, 추상화, 상속, 다형성의 차이를 자주 묻습니다.
- 특히 캡슐화는 “묶는다”, 정보 은닉은 “감춘다/접근을 제한한다”로 구분해두면 좋습니다.

2. 자주 나오는 포인트
- 캡슐화: 데이터와 메서드를 하나의 단위로 묶는 것
- 정보 은닉: 내부 구현을 숨기고 필요한 인터페이스만 공개하는 것

오늘 주제: 프로그래밍 기초 - 예외 처리

1. 핵심 요약
- 예외 처리는 프로그램 실행 중 발생하는 오류나 비정상 상황을 다루는 기법입니다.
- 예를 들어 파일이 없거나, 0으로 나누거나, 네트워크 연결이 끊기는 상황이 예외가 될 수 있습니다.
- try 블록에는 예외가 발생할 수 있는 코드를 둡니다.
- catch 또는 except 블록에서는 발생한 예외를 잡아 적절히 처리합니다.
- finally 블록은 예외 발생 여부와 관계없이 실행되며, 파일 닫기나 DB 연결 해제처럼 자원 정리에 자주 사용됩니다.
- 예외 처리는 프로그램을 무조건 계속 실행시키기 위한 장치가 아니라, 오류 상황을 안전하게 제어하기 위한 구조입니다.
- 모든 예외를 대충 잡고 무시하면 실제 문제 원인이 숨겨져 더 큰 장애로 이어질 수 있습니다.
- 시험에서는 try-catch-finally의 역할, 예외 전파, 자원 해제 목적을 자주 묻습니다.

2. 자주 나오는 포인트
- try: 예외 발생 가능 코드, catch/except: 예외 처리 코드, finally: 무조건 수행되는 정리 코드
- 예외 처리는 오류 은폐가 아니라 안정적인 흐름 제어와 복구를 위한 기법
- 너무 넓은 범위의 예외를 무조건 무시하는 것은 좋은 설계가 아님

3. OX 문제
예외 처리는 프로그램 실행 중 발생한 오류나 비정상 상황을 처리해 프로그램이 무조건 즉시 종료되지 않도록 돕지만, 모든 예외를 무조건 무시하고 계속 실행하는 것이 좋은 설계는 아니다.

지난 문제 정답

정답: O

해설: HTTPS는 HTTP 메시지를 TLS로 보호해 전송 구간의 도청과 변조 위험을 줄입니다. TLS는 암호화로 기밀성을, 메시지 무결성 검증으로 변조 탐지를, 인증서 기반 확인으로 서버 인증을 제공합니다. 시험에서는 HTTPS = HTTP + TLS, 기본 포트 443, HTTP 기본 포트 80과의 차이를 함께 기억하면 좋습니다.

[정보처리기사/전산직 필기]
오늘 주제: 프로그래밍 기초 - 포인터와 메모리 주소

1. 핵심 요약
- 포인터는 “값”이 아니라 “메모리 주소”를 저장하는 변수입니다.
- 예를 들어 일반 변수 a가 실제 데이터 10을 담는다면, 포인터 p는 a가 저장된 위치의 주소를 담을 수 있습니다.
- C 언어 기준으로 &a는 a의 주소를 구하는 연산이고, *p는 p가 가리키는 주소의 값을 접근하는 역참조입니다.
- 포인터를 쓰면 배열, 문자열, 동적 메모리, 함수 인자 전달을 효율적으로 다룰 수 있습니다.
- 하지만 초기화하지 않은 포인터, NULL 포인터 역참조, 이미 해제한 메모리를 다시 참조하는 dangling pointer는 오류를 만들 수 있습니다.
- 특히 포인터는 메모리에 직접 접근하므로 잘못 쓰면 프로그램 비정상 종료나 보안 취약점으로 이어질 수 있습니다.
- 시험에서는 포인터의 개념 자체보다 주소 연산자, 역참조, NULL, 배열과 포인터 관계를 자주 묻습니다.

2. 자주 나오는 포인트
- 포인터 변수는 데이터 값 자체가 아니라 메모리 주소를 저장한다.
- &는 주소를 구할 때, *는 선언 또는 역참조 문맥에서 쓰인다.
- 잘못된 포인터 접근은 런타임 오류, 메모리 손상, 보안 문제로 연결될 수 있다.

3. OX 문제
포인터 변수는 값 자체를 저장하는 일반 변수와 달리 다른 변수나 메모리 공간의 주소를 저장하며, 잘못 사용하면 잘못된 메모리 접근 문제가 발생할 수 있다.
