### 2026-04-27 12:00 - NCS 정보통신 - 다중화 방식

[정보처리기사/전산직 필기]
오늘 주제: NCS 정보통신 - 다중화 방식

1. 핵심 요약
- 다중화는 하나의 전송 매체를 여러 통신 채널이 함께 사용하도록 만드는 기술입니다.
- 회선이나 주파수 자원은 한정되어 있으므로, 여러 데이터를 효율적으로 실어 보내기 위해 사용합니다.
- 대표 방식에는 주파수분할 다중화(FDM), 시분할 다중화(TDM), 파장분할 다중화(WDM), 코드분할 다중화(CDM)가 있습니다.
- FDM은 전체 주파수 대역을 여러 개로 나누어 각 신호에 서로 다른 주파수 대역을 배정합니다.
- TDM은 시간을 잘게 나누고, 각 채널이 정해진 시간 슬롯에 번갈아 데이터를 보냅니다.
- WDM은 광통신에서 서로 다른 빛의 파장을 이용해 여러 신호를 동시에 전송하는 방식입니다.
- CDM은 각 사용자에게 고유한 코드를 부여해 같은 주파수 대역을 공유하게 하는 방식입니다.
- 시험에서는 “무엇을 나누는가?”를 기준으로 구분하면 헷갈림이 줄어듭니다.

2. 자주 나오는 포인트
- FDM: 주파수 분할, 아날로그 통신·방송 계열과 자주 연결
- TDM: 시간 슬롯 분할, 디지털 전송에서 자주 출제
- WDM: 광섬유 통신, 파장 분할이 핵심 키워드

3. OX 문제
시분할 다중화(TDM)는 여러 신호가 같은 전송 매체를 사용할 때 시간 슬롯을 나누어 번갈아 전송하는 방식이다.

### 2026-05-06 16:00 - NCS 정보통신 - 주파수 분할 다중화 FDM

정답: O

해설: 대칭키 암호화는 암호화와 복호화에 같은 비밀키를 사용하는 방식입니다. AES처럼 처리 속도가 빠른 편이라 대량 데이터 암호화에 자주 쓰입니다. 다만 송신자와 수신자가 같은 키를 안전하게 공유해야 하므로 키 분배와 키 관리가 중요합니다. 실제 HTTPS 같은 환경에서는 공개키 방식으로 세션 키를 안전하게 합의한 뒤, 실제 데이터는 대칭키로 빠르게 암호화하는 혼합 방식을 많이 사용합니다.

[정보처리기사/전산직 필기]
오늘 주제: NCS 정보통신 - 주파수 분할 다중화 FDM

1. 핵심 요약
- 다중화는 하나의 전송 매체를 여러 사용자가 함께 쓰도록 신호를 묶어 보내는 기술입니다.
- FDM, 즉 주파수 분할 다중화는 전체 주파수 대역을 여러 개의 작은 대역으로 나누는 방식입니다.
- 각 채널은 서로 다른 주파수 대역을 배정받기 때문에 여러 신호를 동시에 전송할 수 있습니다.
- 라디오 방송에서 방송국마다 다른 주파수를 사용하는 것을 떠올리면 이해하기 쉽습니다.
- 인접한 주파수 대역끼리 간섭이 생기지 않도록 보호 대역을 둘 수 있습니다.
- 다만 보호 대역은 실제 데이터 전송에는 쓰이지 않으므로 대역폭 효율이 조금 떨어질 수 있습니다.
- TDM은 시간을 나눠 번갈아 보내는 방식이고, FDM은 주파수를 나눠 동시에 보내는 방식이라는 차이가 핵심입니다.
- 광통신에서는 주파수 대신 빛의 파장을 나누는 WDM도 함께 비교해 자주 출제됩니다.

2. 자주 나오는 포인트
- FDM: 주파수 대역 분할, 여러 채널 동시 전송
- TDM: 시간 슬롯 분할, 채널들이 번갈아 전송
- 보호 대역: 채널 간 간섭 방지 목적이지만 대역폭 낭비 가능

3. OX 문제
주파수 분할 다중화(FDM)는 하나의 전송 매체를 여러 주파수 대역으로 나누어 여러 신호를 동시에 전송하는 방식이다.

### 2026-05-09 20:00 - NCS 정보통신 - 광통신 WDM 파장 분할 다중화

지난 문제 정답

정답: O

해설: 공유락(S Lock)은 여러 트랜잭션이 같은 데이터를 동시에 읽는 것을 허용할 수 있습니다. 반면 배타락(X Lock)은 데이터를 변경할 때 사용하는 잠금이라, 다른 트랜잭션의 읽기나 쓰기를 제한해 데이터 불일치를 막습니다. 시험에서는 공유락=읽기 중심, 배타락=쓰기 중심, 그리고 로킹은 동시성 제어 기법이라는 점을 연결해서 기억하면 좋습니다.

[정보처리기사/전산직 필기]
오늘 주제: NCS 정보통신 - 광통신 WDM 파장 분할 다중화

1. 핵심 요약
- WDM은 Wavelength Division Multiplexing, 즉 파장 분할 다중화입니다.
- 광섬유 통신에서 서로 다른 파장의 빛을 각각 하나의 채널처럼 사용합니다.
- 하나의 광섬유에 여러 파장의 신호를 동시에 실어 보낼 수 있어 전송 용량을 크게 늘릴 수 있습니다.
- 전기 통신의 FDM이 주파수 대역을 나누는 방식이라면, WDM은 광통신에서 빛의 파장을 나누는 방식입니다.
- 예를 들어 빨간빛, 파란빛처럼 서로 다른 파장의 빛을 같은 광섬유에 함께 보내고, 수신 측에서 다시 분리한다고 이해하면 쉽습니다.
- 광섬유 인프라를 새로 많이 깔지 않고도 회선 용량을 확장할 수 있다는 장점이 있습니다.
- 정보통신/NCS에서는 다중화 방식 비교 문제가 자주 나오므로 TDM, FDM, WDM을 함께 구분해야 합니다.

2. 자주 나오는 포인트
- WDM: 광통신, 파장 분할, 여러 광 신호 동시 전송
- FDM: 주파수 대역 분할 / TDM: 시간 슬롯 분할
- WDM은 광섬유의 전송 용량 확대와 관련된 기술로 자주 출제됩니다.

3. OX 문제
WDM은 광섬유에서 서로 다른 파장의 빛을 각각의 채널처럼 사용해 여러 신호를 동시에 전송하는 다중화 방식이다.

### 2026-05-13 19:00 - NCS 정보통신 - WDM과 TDM 다중화 비교

지난 문제 정답

정답: O

해설: 퀵 정렬은 피벗을 기준으로 데이터를 나누는 분할 정복 정렬입니다. 평균적으로는 분할이 균형 있게 이루어져 O(n log n)이지만, 피벗이 계속 최솟값이나 최댓값처럼 한쪽으로 치우치면 한쪽 부분 배열이 거의 n-1 크기로 남습니다. 이 경우 비교 횟수가 크게 늘어 최악 시간복잡도는 O(n²)이 될 수 있습니다.

[정보처리기사/전산직 필기]
오늘 주제: NCS 정보통신 - WDM과 TDM 다중화 비교

1. 핵심 요약
- 다중화는 하나의 전송 매체를 여러 신호가 함께 사용하도록 하는 기술입니다.
- TDM은 시간 분할 다중화로, 시간을 여러 슬롯으로 나누어 각 신호가 번갈아 전송됩니다.
- 예를 들어 A, B, C 신호가 있으면 A 시간, B 시간, C 시간처럼 순서를 나누어 매체를 공유합니다.
- FDM은 주파수 대역을 나누어 여러 신호를 동시에 보내는 방식입니다.
- WDM은 광통신에서 쓰는 파장 분할 다중화로, 서로 다른 파장의 빛을 각각 채널처럼 사용합니다.
- WDM은 하나의 광섬유에 여러 파장을 동시에 실을 수 있어 전송 용량을 크게 높일 수 있습니다.
- 쉽게 말해 TDM은 “시간을 나눠 쓰기”, WDM은 “빛의 색깔/파장을 나눠 쓰기”로 이해하면 좋습니다.

2. 자주 나오는 포인트
- TDM: 시간 슬롯 분할, 번갈아 전송
- FDM: 주파수 대역 분할, 동시 전송
- WDM: 광섬유, 파장 분할, 대용량 동시 전송

3. OX 문제
광통신에서 WDM은 여러 파장의 빛을 하나의 광섬유에 함께 실어 보내 전송 용량을 높이는 방식이며, TDM은 시간 슬롯을 나누어 신호를 번갈아 전송하는 방식이다.

지난 문제 정답
정답: O

해설: INSERT INTO ... SELECT는 SELECT 결과로 나온 여러 행을 다른 테이블에 한 번에 삽입할 수 있는 DML입니다. VALUES로 직접 값을 넣는 방식과 달리, 조회 결과를 이용해 백업 테이블이나 집계 테이블에 데이터를 넣을 때 활용됩니다. 단, SELECT 결과 컬럼 개수와 타입이 INSERT 대상 컬럼과 맞아야 하고 제약조건도 만족해야 합니다.

[정보처리기사/전산직 필기]
오늘 주제: NCS 정보통신 - PCM 표본화·양자화·부호화

1. 핵심 요약
- PCM은 아날로그 신호를 디지털 신호로 변환하는 대표적인 방식입니다.
- 음성처럼 연속적으로 변하는 신호를 컴퓨터나 디지털 통신망에서 처리하려면 0과 1로 바꿔야 합니다.
- 첫 단계는 표본화로, 연속 신호를 일정한 시간 간격으로 측정합니다.
- 두 번째는 양자화로, 측정된 값을 정해진 단계 중 가장 가까운 값으로 근사합니다.
- 이때 원래 값과 근사값 사이에 양자화 오차가 생길 수 있습니다.
- 마지막은 부호화로, 양자화된 값을 이진수 코드로 표현합니다.
- 흐름은 “표본화 → 양자화 → 부호화” 순서로 외우면 좋습니다.

2. 자주 나오는 포인트
- PCM 순서: 표본화 → 양자화 → 부호화
- 양자화 단계에서는 오차가 발생할 수 있음
- 아날로그 신호를 디지털 데이터로 바꾸는 방식이라는 큰 목적을 기억하기

3. OX 문제
PCM은 아날로그 신호를 디지털 신호로 바꾸기 위해 표본화, 양자화, 부호화 과정을 거친다.
