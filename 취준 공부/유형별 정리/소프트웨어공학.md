오늘 주제: 소프트웨어공학 - 결합도와 응집도

지난 문제 정답
정답: X

해설: 제3정규형(3NF)은 2NF를 만족한 뒤, 이행적 함수 종속까지 제거해야 합니다. 즉 기본키가 아닌 속성이 다른 기본키가 아닌 속성을 통해 간접적으로 결정되면 안 됩니다. 부분 함수 종속 제거만으로는 2NF 수준입니다.

1. 핵심 요약
- 모듈화는 큰 소프트웨어를 기능 단위로 나누어 이해와 수정이 쉽도록 만드는 설계 기법입니다.
- 좋은 모듈 설계의 핵심 기준은 응집도는 높게, 결합도는 낮게 만드는 것입니다.
- 응집도는 한 모듈 안의 구성 요소들이 하나의 목적을 위해 얼마나 밀접하게 관련되어 있는지를 뜻합니다.
- 응집도가 높으면 모듈이 맡은 역할이 분명해져 수정 범위가 작아지고 재사용도 쉬워집니다.
- 결합도는 모듈과 모듈 사이의 의존 정도를 의미합니다.
- 결합도가 높으면 한 모듈의 변경이 다른 모듈에 연쇄적으로 영향을 주기 쉬워 유지보수가 어려워집니다.
- 그래서 설계할 때는 내부적으로는 관련 기능을 모으고, 외부와는 필요한 인터페이스만 통해 느슨하게 연결하는 것이 좋습니다.

2. 자주 나오는 포인트
- 좋은 설계 원칙: 고응집, 저결합
- 응집도는 기능적 응집이 가장 좋고, 우연적 응집이 가장 나쁩니다.
- 결합도는 자료 결합이 비교적 좋고, 내용 결합이 가장 나쁩니다.

3. OX 문제
좋은 모듈 설계에서는 모듈 내부 기능들이 서로 밀접하게 관련될수록 응집도가 높고, 다른 모듈에 덜 의존할수록 결합도가 낮다.

### 2026-05-02 13:00 - 소프트웨어공학 - 자료 흐름도 DFD

[정보처리기사/전산직 필기]
오늘 주제: 소프트웨어공학 - 자료 흐름도 DFD

1. 핵심 요약
- 자료 흐름도(DFD, Data Flow Diagram)는 시스템 안에서 데이터가 어떻게 이동하고 처리되는지 표현하는 분석 도구입니다.
- “누가 데이터를 입력하는가 → 어떤 처리를 거치는가 → 어디에 저장되는가 → 어디로 출력되는가”를 그림으로 정리합니다.
- 주요 구성 요소는 프로세스, 데이터 흐름, 데이터 저장소, 단말/외부 엔터티입니다.
- 프로세스는 입력 데이터를 출력 데이터로 변환하는 처리 기능을 의미합니다.
- 데이터 저장소는 파일, DB 테이블처럼 데이터가 보관되는 장소입니다.
- 외부 엔터티는 시스템 밖에서 데이터를 주고받는 사람, 조직, 다른 시스템입니다.
- DFD는 프로그램 실행 순서나 조건 분기보다 데이터의 이동과 변환에 초점을 둡니다.
