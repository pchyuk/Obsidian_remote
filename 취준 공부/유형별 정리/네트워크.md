### 2026-04-27 14:00 - 네트워크 - DNS와 이름 해석

[정보처리기사/전산직 필기]
오늘 주제: 네트워크 - DNS와 이름 해석

1. 핵심 요약
- DNS는 Domain Name System의 약자로, 사람이 기억하기 쉬운 도메인 이름을 실제 통신에 필요한 IP 주소로 바꿔 주는 시스템입니다.
- 예를 들어 `www.example.com`에 접속하면 컴퓨터는 결국 해당 서버의 IP 주소를 알아야 패킷을 보낼 수 있습니다.
- DNS는 하나의 중앙 서버가 전부 처리하는 방식이 아니라, 루트 DNS → TLD DNS → 권한 있는 DNS 서버처럼 계층적으로 동작합니다.
- 루트 DNS는 `.com`, `.kr` 같은 최상위 도메인 서버 위치를 알려 주고, TLD DNS는 해당 도메인을 관리하는 서버를 안내합니다.
- 권한 있는 DNS 서버는 최종적으로 특정 도메인에 대응하는 IP 주소 같은 레코드를 제공합니다.
- 자주 쓰는 DNS 결과는 캐시에 저장됩니다. 그래서 같은 사이트에 다시 접속할 때 매번 전체 조회 과정을 반복하지 않아도 됩니다.
- DNS 레코드에는 IPv4 주소를 나타내는 A 레코드, IPv6 주소를 나타내는 AAAA 레코드, 별칭을 나타내는 CNAME, 메일 서버를 나타내는 MX 등이 있습니다.
- 시험에서는 “도메인 이름 ↔ IP 주소 변환”, “계층적 분산 구조”, “캐싱”, “주요 레코드 종류”를 함께 묶어 기억하면 좋습니다.

2. 자주 나오는 포인트
- DNS는 도메인 이름을 IP 주소로 변환하는 이름 해석 시스템입니다.
- 루트 DNS, TLD DNS, 권한 있는 DNS 서버로 이어지는 계층 구조를 사용합니다.
- 캐싱은 DNS 조회 속도를 높이고 네트워크 부하를 줄이지만, TTL 동안 오래된 정보가 남을 수도 있습니다.

3. OX 문제
DNS는 사람이 읽기 쉬운 도메인 이름을 IP 주소로 변환해 주는 시스템이며, 일반적으로 계층적 구조와 캐싱을 이용해 조회 효율을 높인다.

### 2026-04-27 19:00 - 네트워크 - IP 주소와 서브넷 마스크

[정보처리기사/전산직 필기]
오늘 주제: 네트워크 - IP 주소와 서브넷 마스크

1. 핵심 요약
- IP 주소는 네트워크에서 장비를 식별하기 위한 논리적 주소입니다.
- IPv4는 보통 `192.168.0.10`처럼 10진수 4개로 표현하지만, 내부적으로는 32비트입니다.
- 서브넷 마스크는 IP 주소 중 어디까지가 “네트워크 부분”이고 어디부터가 “호스트 부분”인지 구분하는 기준입니다.
- 예를 들어 `/24`는 앞 24비트가 네트워크 부분이고, 나머지 8비트가 호스트 부분이라는 뜻입니다.
- 같은 네트워크 주소를 가진 장비들은 같은 서브넷에 있다고 봅니다.
- 같은 서브넷 안에서는 보통 라우터를 거치지 않고 직접 통신할 수 있습니다.
- 다른 서브넷으로 가려면 기본 게이트웨이, 즉 라우터를 통해 패킷이 전달됩니다.
- 그래서 IP 주소와 서브넷 마스크를 함께 봐야 “같은 네트워크인지, 다른 네트워크인지” 판단할 수 있습니다.
- 시험에서는 `/24`, `/26` 같은 CIDR 표기와 사용 가능한 호스트 수 계산이 자주 나옵니다.

2. 자주 나오는 포인트
- 서브넷 마스크: 네트워크 부분과 호스트 부분을 구분하는 값
- 같은 서브넷 여부: IP 주소에 서브넷 마스크를 적용한 네트워크 주소가 같은지 확인
- 호스트 수 계산: 호스트 비트가 n개면 사용 가능 호스트 수는 보통 `2ⁿ - 2`

3. OX 문제
서브넷 마스크는 IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 기준이며, 같은 서브넷에 속한 장비들은 일반적으로 라우터를 거치지 않고 직접 통신할 수 있다.

오늘 주제: 네트워크 - 라우팅과 라우터

1. 핵심 요약

- 라우터는 서로 다른 네트워크를 연결하고, 목적지 IP 주소를 기준으로 다음에 보낼 경로를 결정합니다.
- 이때 참고하는 표가 라우팅 테이블이며, 목적지 네트워크, 서브넷 마스크, 게이트웨이, 인터페이스 등의 정보가 들어갑니다.

- 라우터는 목적지까지 직접 보내는 것이 아니라, 보통 “다음 홉”으로 패킷을 넘기는 식으로 단계적으로 전달합니다.
- 라우팅 방식은 관리자가 직접 경로를 넣는 정적 라우팅과, 라우팅 프로토콜이 자동으로 경로를 갱신하는 동적 라우팅으로 나눌 수 있습니다.
- 시험에서는 라우터가 OSI 3계층 장비이고, IP 주소 기반으로 경로 선택을 한다는 점이 자주 나옵니다.
- 네트워크 문제에서 “서로 다른 네트워크 간 통신”, “게이트웨이”, “최적 경로 선택”이 보이면 라우팅을 떠올리면 됩니다.

2. 자주 나오는 포인트
- 라우터: 네트워크 계층 장비, IP 주소 기반으로 패킷 전달
- 스위치와 구분: 스위치는 주로 같은 LAN 내부에서 MAC 주소 기반 전달
- 정적 라우팅은 단순하지만 관리 부담이 있고, 동적 라우팅은 변화 대응이 좋지만 프로토콜 이해가 필요

3. OX 문제
라우터는 목적지 IP 주소와 라우팅 테이블을 참고해 패킷을 다음 홉으로 전달하며, 서로 다른 네트워크 간 통신을 가능하게 한다.

### 2026-04-28 15:00 - 네트워크 - HTTP와 무상태성

[정보처리기사/전산직 필기]
오늘 주제: 네트워크 - HTTP와 무상태성

1. 핵심 요약
- HTTP는 웹 브라우저 같은 클라이언트와 웹 서버가 데이터를 주고받기 위한 응용 계층 프로토콜입니다.
- 기본 구조는 클라이언트가 요청(Request)을 보내고, 서버가 응답(Response)을 돌려주는 방식입니다.
- 예를 들어 브라우저가 “이 페이지 보여줘”라고 요청하면 서버는 HTML, 이미지, JSON 같은 데이터를 응답합니다.
- HTTP의 중요한 특징은 무상태성(stateless)입니다. 즉, 서버는 기본적으로 이전 요청을 자동으로 기억하지 않습니다.
- 그래서 로그인 유지, 장바구니, 사용자 설정처럼 “이전 상태”가 필요한 기능은 쿠키, 세션, 토큰 등을 함께 사용합니다.
- HTTP는 평문 통신이고, HTTPS는 HTTP에 SSL/TLS 암호화를 적용해 기밀성과 무결성을 강화한 방식입니다.
- 동작 원리는 보통 TCP 연결 위에서 요청 메시지와 응답 메시지를 주고받는 형태로 이해하면 됩니다.
- 시험에서는 HTTP가 어느 계층인지, 요청/응답 구조, 무상태성, HTTPS와의 차이가 자주 나옵니다.

2. 자주 나오는 포인트
- HTTP: 응용 계층 프로토콜, 요청/응답 방식
- 무상태성: 이전 요청 정보를 기본적으로 기억하지 않음
- HTTPS: HTTP + SSL/TLS, 암호화로 보안 강화

3. OX 문제
HTTP는 웹 클라이언트와 서버가 요청과 응답을 주고받기 위한 응용 계층 프로토콜이며, 기본적으로 각 요청은 독립적으로 처리되므로 상태 유지를 위해 쿠키나 세션 같은 보조 수단을 사용할 수 있다.

오늘 주제: 네트워크 - TCP 흐름 제어와 혼잡 제어

1. 핵심 요약
- 흐름 제어는 송신자가 수신자의 처리 능력을 넘어서 데이터를 보내지 않도록 조절하는 기능입니다.
- 쉽게 말해 “받는 쪽 버퍼가 넘치지 않게 하는 제어”입니다.
- TCP에서는 수신자가 받을 수 있는 버퍼 크기를 수신 윈도우, rwnd로 알려 줍니다.
- 송신자는 이 값을 보고 한 번에 보낼 수 있는 데이터 양을 조절합니다.
- 혼잡 제어는 수신자 하나의 문제가 아니라, 네트워크 경로 전체가 막히지 않도록 전송량을 조절하는 기능입니다.
- 라우터나 회선에 패킷이 몰리면 지연, 손실, 재전송이 늘어나므로 TCP는 전송 속도를 줄이거나 천천히 늘립니다.
- 대표 개념으로 혼잡 윈도우(cwnd), 슬로 스타트, 혼잡 회피, 빠른 재전송이 자주 나옵니다.
- 정리하면 흐름 제어는 수신자 보호, 혼잡 제어는 네트워크 보호입니다.

2. 자주 나오는 포인트
- 흐름 제어: 수신 버퍼 초과 방지, 수신 윈도우(rwnd), 송신자-수신자 간 속도 조절
- 혼잡 제어: 네트워크 과부하 방지, 혼잡 윈도우(cwnd), 슬로 스타트/혼잡 회피
- 시험에서는 “흐름 제어 = 수신자 기준”, “혼잡 제어 = 네트워크 상태 기준” 구분이 핵심입니다.

3. OX 문제
TCP의 흐름 제어는 수신자의 처리 능력을 넘지 않도록 송신량을 조절하는 것이고, 혼잡 제어는 네트워크 내부의 혼잡을 줄이기 위해 전송 속도를 조절하는 것이다.

오늘 주제: 네트워크 - OSI 7계층과 TCP/IP 모델

1. 핵심 요약
- OSI 7계층은 네트워크 통신 기능을 역할별로 나눈 참조 모델입니다.
- 아래 계층은 실제 전송에 가깝고, 위 계층은 사용자 서비스에 가깝습니다.
- 1계층 물리 계층은 전기 신호, 케이블, 허브처럼 비트 전송을 다룹니다.
- 2계층 데이터링크 계층은 같은 네트워크 안에서 프레임 전달과 MAC 주소를 다룹니다.
- 3계층 네트워크 계층은 IP 주소와 라우팅을 통해 다른 네트워크로 패킷을 보냅니다.
- 4계층 전송 계층은 TCP/UDP처럼 종단 간 데이터 전달을 담당합니다.
- TCP/IP 모델은 실무 인터넷 구조에 가까운 모델이고, OSI 모델은 시험에서 계층 구분용으로 자주 나옵니다.
- 핵심은 “프로토콜이 어느 계층인가”를 정확히 매칭하는 것입니다.

2. 자주 나오는 포인트
- TCP/UDP: 전송 계층, IP: 네트워크 계층, Ethernet: 데이터링크 계층
- HTTP, FTP, SMTP, DNS: 응용 계층으로 자주 출제
- 장비 매칭: 허브 1계층, 스위치 2계층, 라우터 3계층

3. OX 문제
OSI 7계층은 네트워크 기능을 계층별로 나눈 모델이며, TCP와 UDP는 전송 계층, IP는 네트워크 계층, HTTP와 DNS는 응용 계층에 해당한다.

### 2026-04-29 11:00 - 네트워크 - CIDR과 사용 가능 호스트 수 계산

[정보처리기사/전산직 필기]
오늘 주제: 네트워크 - CIDR과 사용 가능 호스트 수 계산

1. 핵심 요약
- CIDR은 IP 주소 뒤에 `/24`, `/26`처럼 붙는 표기로, 네트워크 부분이 몇 비트인지 나타냅니다.
- IPv4 주소는 전체 32비트이므로 `/26`이면 앞 26비트는 네트워크, 뒤 6비트는 호스트 부분입니다.
- 호스트 비트가 n개이면 전체 주소 수는 `2^n`개입니다.
- 일반적인 서브넷에서는 첫 주소는 네트워크 주소, 마지막 주소는 브로드캐스트 주소로 예약됩니다.
- 그래서 사용 가능한 호스트 수는 보통 `2^호스트비트 - 2`로 계산합니다.

- 일반적으로 ARP 응답은 요청한 장비에게 유니캐스트로 돌아옵니다.

- IPv6는 IPv4 주소 부족 문제를 해결하기 위해 도입된 128비트 IP 주소 체계입니다.
- IPv4는 32비트라 주소 개수가 제한적이지만, IPv6는 훨씬 넓은 주소 공간을 제공합니다.
- IPv6 주소는 보통 16비트씩 8개 그룹을 콜론(:)으로 구분해 16진수로 표현합니다.
- 예: 2001:0db8:0000:0000:0000:ff00:0042:8329 같은 형태입니다.
- 연속된 0 그룹은 한 번만 ::로 축약할 수 있어 주소를 짧게 쓸 수 있습니다.
- IPv6는 브로드캐스트를 사용하지 않고, 멀티캐스트와 애니캐스트를 활용합니다.
- 주소 자동 설정 기능을 지원해 장비가 네트워크에 접속할 때 주소 설정을 더 쉽게 할 수 있습니다.
- 시험에서는 “IPv4는 32비트, IPv6는 128비트” 구분이 가장 먼저 나옵니다.

2. 자주 나오는 포인트
- IPv6 주소 길이: 128비트, IPv4는 32비트
- IPv6 표기: 콜론(:)으로 구분한 16진수 8그룹
- IPv6 특징: 주소 공간 확대, 브로드캐스트 미사용, 자동 설정 지원

3. OX 문제
IPv6는 128비트 주소 체계를 사용하며, 주소 공간이 IPv4보다 훨씬 커서 IPv4 주소 부족 문제를 완화하기 위해 도입되었다.

해설: 식별자는 엔터티 인스턴스, 즉 개별 행을 유일하게 구분하는 속성 또는 속성 집합입니다. 여러 후보 식별자 중 대표로 선택한 것이 주식별자이고, 관계형 모델에서는 보통 기본키(PK)로 이어집니다. SQLD/정처기 모두 식별자의 유일성, 최소성, 불변성, 존재성과 후보키·기본키·대체키 관계를 자주 묻습니다.

오늘 주제: 네트워크 - 스위치와 MAC 주소 테이블

1. 핵심 요약
- 스위치는 주로 데이터 링크 계층, 즉 2계층에서 동작하는 네트워크 장비입니다.
- 핵심 기준은 IP 주소가 아니라 MAC 주소입니다.
- 스위치는 프레임이 들어오면 출발지 MAC 주소와 들어온 포트를 MAC 주소 테이블에 기록합니다.
- 이후 목적지 MAC 주소가 테이블에 있으면 해당 포트로만 프레임을 전달합니다.
- 목적지 MAC 주소를 모르면 같은 VLAN 안의 다른 포트들로 프레임을 플러딩할 수 있습니다.
- 허브는 들어온 신호를 모든 포트로 단순 전달하지만, 스위치는 MAC 주소를 학습해 더 효율적으로 전달합니다.
- 그래서 스위치는 포트별 충돌 영역을 나누어 네트워크 효율을 높입니다.
- 라우터는 3계층에서 IP 주소와 라우팅 테이블을 보고 다른 네트워크로 패킷을 전달한다는 점에서 스위치와 구분됩니다.

2. 자주 나오는 포인트
- 스위치: 2계층, MAC 주소 기반, 프레임 전달
- 라우터: 3계층, IP 주소 기반, 서로 다른 네트워크 연결
- 목적지 MAC을 모르면 플러딩, 알면 해당 포트로 선택 전달

3. OX 문제
스위치는 MAC 주소 테이블을 이용해 목적지 MAC 주소가 연결된 포트로 프레임을 전달하며, 목적지 MAC을 모르면 플러딩할 수 있다.
