### 2026-04-26 18:30 - 운영체제 - CPU 스케줄링

찬혁 님, 오늘은 **운영체제 - CPU 스케줄링**입니다.

CPU 스케줄링은 여러 프로세스 중 어떤 프로세스에게 CPU를 배정할지 결정하는 운영체제의 핵심 기능입니다.  
스케줄링 방식은 크게 **선점형**과 **비선점형**으로 나뉩니다.  
비선점형은 한 프로세스가 CPU를 잡으면 종료되거나 대기 상태가 될 때까지 CPU를 빼앗지 않습니다.  
반면 선점형은 타이머 인터럽트나 더 높은 우선순위 프로세스 등장 시 CPU를 강제로 회수할 수 있습니다.  
대표적인 비선점형 방식에는 FCFS, SJF, HRN이 있고, 선점형 방식에는 Round Robin, SRT, 선점 우선순위 스케줄링이 있습니다.  
공기업 전산직 필기에서는 각 알고리즘의 특징과 평균 대기시간 계산 문제가 자주 나옵니다.

자주 나오는 포인트:
- **Round Robin**은 시간 할당량이 핵심이며, 너무 작으면 문맥 교환 오버헤드가 커집니다.
- **SJF/SRT**는 실행 시간이 짧은 작업을 우선해 평균 대기시간을 줄이는 데 유리합니다.
- **FCFS**는 구현은 단순하지만 convoy effect가 발생할 수 있습니다.

OX 문제:  
**운영체제에서 선점형 스케줄링은 실행 중인 프로세스의 CPU를 강제로 회수해 다른 프로세스에 배정할 수 있다.**

정답: O

해설: FOREIGN KEY(외래키)는 자식 테이블의 컬럼 값이 부모 테이블의 기본키 또는 유일키 값을 참조하도록 제한하는 제약조건입니다. 예를 들어 주문 테이블의 고객ID가 고객 테이블에 존재하는 고객ID만 가질 수 있게 하면, 존재하지 않는 고객의 주문이 저장되는 문제를 막을 수 있습니다. 이것이 참조 무결성입니다.

[정보처리기사/전산직 필기]
오늘 주제: 운영체제 - 페이지 교체 알고리즘 FIFO와 LRU

1. 핵심 요약
- 가상메모리에서는 필요한 페이지만 메모리에 올리고, 부족하면 기존 페이지 중 하나를 내보냅니다.
- 이때 어떤 페이지를 교체할지 정하는 기준이 페이지 교체 알고리즘입니다.
- FIFO는 First In First Out으로, 메모리에 가장 먼저 들어온 페이지를 먼저 교체합니다.
- 구현은 단순하지만, 오래된 페이지가 자주 쓰이는 페이지일 수도 있어 비효율이 생길 수 있습니다.
- LRU는 Least Recently Used로, 가장 오랫동안 사용되지 않은 페이지를 교체합니다.
- “최근에 안 쓴 페이지는 앞으로도 덜 쓸 가능성이 높다”는 지역성 원리를 활용합니다.
- 시험에서는 FIFO와 LRU의 기준 차이를 묻거나, 참조 문자열을 주고 페이지 부재 횟수를 계산하게 합니다.
- 계산 문제는 각 참조 시점마다 메모리 프레임 상태를 차례대로 적으면 실수가 줄어듭니다.

2. 자주 나오는 포인트
- FIFO: “먼저 들어온 페이지”를 교체한다.
- LRU: “가장 오래 사용되지 않은 페이지”를 교체한다.
- 페이지 부재 횟수 계산 문제에서는 적재 시점과 최근 사용 시점을 혼동하지 않는 것이 핵심입니다.

3. OX 문제
LRU 페이지 교체 알고리즘은 메모리에 가장 오래 머무른 페이지를 먼저 교체하는 방식이다.

- 세마포어는 동기화 도구지만 잘못된 순서로 사용하면 교착상태가 생길 수 있음.

3. OX 문제

- SRT: SJF의 선점형 변형으로, “남은 실행 시간”이 기준이다.

오늘 주제: 운영체제 - 동적 메모리 할당 방식

1. 핵심 요약
- 동적 메모리 할당은 프로세스가 필요로 하는 크기만큼 메모리의 빈 공간에 배치하는 방식입니다.
- 메모리에 여러 프로세스가 들어오고 나가면 빈 공간이 조각조각 나뉘는 외부 단편화가 생길 수 있습니다.
- 최초 적합(First Fit)은 앞에서부터 탐색하다가 들어갈 수 있는 첫 번째 빈 공간에 배치합니다.
- 최적 적합(Best Fit)은 들어갈 수 있는 공간 중 가장 작은 빈 공간을 골라 남는 공간을 최소화하려 합니다.
- 최악 적합(Worst Fit)은 가장 큰 빈 공간에 배치해 남는 공간도 크게 유지하려는 방식입니다.
- 최초 적합은 빠르지만 앞쪽에 작은 단편이 쌓일 수 있고, 최적 적합은 탐색 비용과 작은 단편 문제가 생길 수 있습니다.
- 시험에서는 “무엇을 기준으로 빈 공간을 고르는가”를 구분하는 문제가 자주 나옵니다.

2. 자주 나오는 포인트
- 최초 적합 = 처음 만나는 충분한 공간
- 최적 적합 = 조건을 만족하는 공간 중 가장 작은 공간
- 최악 적합 = 가장 큰 공간 선택, 외부 단편화 개념과 함께 출제

3. OX 문제
운영체제의 동적 메모리 할당에서 최초 적합은 사용 가능한 빈 공간 중 크기가 가장 작은 공간을 항상 찾아 배치하는 방식이다.
